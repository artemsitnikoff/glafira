# ТЗ-5. Вакансии + Воронка (Экраны 02 + 03)

> **Кому:** React-агент.
> **Зависит от:** ТЗ-3 (каркас, сайдбар уже умеет список вакансий), ТЗ-2 (`/vacancies/*`, `/applications/*`).
> **Источник UX:** прототипы `Vacancies.jsx`, `Candidates.jsx` (`CandidatesList`) + описания Экранов 02, 03.
> **Роуты:** `/vacancies` (empty), `/vacancies/archive`, `/vacancies/new`, `/vacancies/:id` (воронка), `/vacancies/:id/edit`.

> Карточка соискателя (Экран 04), которая выезжает в воронке — отдельный ТЗ-6. Здесь делаем воронку и режим «открыта карточка» как layout, а саму карточку монтируем из ТЗ-6.

## ЧАСТЬ A. Раздел «Вакансии» (Экран 02)

Раздел не имеет отдельной средней колонки — список вакансий живёт в сайдбаре (уже сделан в ТЗ-3). Контентная область справа показывает: empty / воронку / архив / форму создания-редактирования / форму добавления кандидата.

### A.1 Empty state (`/vacancies`, компонент EmptyVacancyPane)
- Иллюстрация-плейсхолдер + заголовок + пояснение «Выберите вакансию слева или создайте новую».
- Показывается, когда раздел открыт, но вакансия не выбрана и архив не активен.

### A.2 Архив (`/vacancies/archive`)
Данные: `GET /vacancies?status=archived`.
Таблица закрытых вакансий:
| Название | Клиент | Рекрутер | Закрыта (дата) | Итог | Срок (дней, моно) |
- Итог: «Нанят»/«Отменена»/«Заморожена» (`archive_result` → лейбл).
- Срок — дни от создания до закрытия.

### A.3 Форма создания/редактирования (`/vacancies/new`, `/vacancies/:id/edit`, компонент NewVacancy)
Многошаговая форма В КОНТЕНТНОЙ ОБЛАСТИ (не модалка). Навигация между разделами (табы/шаги):

**Шаг 1. Описание вакансии**
- Название (text, required).
- **Сортировка** — узкий числовой инпут (моно, шаг 10, дефолт 500), подпись «(порядок в списке)». Определяет позицию в сайдбаре.
- Город, дедлайн (date), число позиций, отдел, тип занятости, конфиденциальность (toggle).
- Вилка ЗП `от/до` + валюта.
- Тело описания (textarea).

**Шаг 2. Воронка**
- Выбор шаблона (radio/select): По умолчанию / Массовый подбор / Техническая с тестовым / Продажи (`default/mass/technical/sales`).
- Редактирование этапов (старт · промежуточные · финальные) — список с возможностью переименовать/добавить. В MVP можно показать этапы шаблона read-only + базовое редактирование лейблов.

**Шаг 3. Команда**
- Мульти-выбор сотрудников (`GET /users` → UserShort). Первый добавленный — ответственный (бейдж «ответственный»).

**Шаг 4. Автоматизация**
- Режим Глафиры (A/B/C) — сегментный контрол с тултипами.
- Автоперевод (`auto_move` toggle + порог-слайдер).
- Автоквалификация (`auto_qa` диапазон этапов от/до).
- Автоотказ (`auto_reject` toggle).

Отправка: `POST /vacancies` (create) или `PATCH /vacancies/:id` (edit). После сохранения → navigate в воронку `/vacancies/:id`. В editMode форма предзаполнена из `GET /vacancies/:id`.

Кнопка submit активна только при заполненном названии.

### A.4 Добавление кандидата (компонент NewCandidate)
Открывается из воронки по «+ Кандидат». Форма в контентной области, привязана к `vacancyId`.

Поля (см. Экран 02 §8):
- «Добавить в вакансию» (dropdown, предзаполнен текущей).
- **Источник** (dropdown, required): HeadHunter / Avito Работа / SuperJob / Telegram-канал / Реферальная программа / Прямой контакт / Кадровое агентство / Другое. Подпись «· откуда узнали о кандидате».
- Фамилия (required) / Имя (required) / Отчество.
- Телефон (masked) / E-mail.
- Пол (segmented) / ДР (date) / Город.
- Ожидаемая ЗП + валюта (`руб.`/`$`/`€`).
- Тип добавления (dropdown: Ручное/Из резюме/Из общей базы/По ссылке HH, дефолт manual).
- Соцсети (тип + url) / Комментарий (textarea).
- Резюме (drop-zone) / Excel-импорт.

Кнопка «+ Добавить кандидата» активна при `Фамилия + Имя + Источник`. Submit → `POST /candidates` (с `vacancy_id`). После → возврат в воронку.

## ЧАСТЬ B. Воронка (Экран 03, `/vacancies/:id`, компонент Funnel)

Главный рабочий экран. Данные шапки: `GET /vacancies/:id` (VacancyDetail). Этапы: `GET /vacancies/:id/stages` (VacancyStageCount[]). Таблица: `GET /vacancies/:id/applications` с фильтрами.

### B.1 Шапка вакансии (vac-header)
**Левая часть:**
- H1: название (`--font-display`, 24–28px).
- Мета-строка через «·»: клиент · рекрутёр · город · «создана {дата}».

**Правая часть — действия:**
1. «Поделиться» (secondary) — копирует ссылку на воронку в буфер.
2. «Перейти на hh» (secondary) — открывает `external_url` (если есть).
3. «Редактировать» (secondary) → `/vacancies/:id/edit`.
4. «+ Добавить кандидата» (primary) → форма NewCandidate.

### B.2 Воронка-чипы (funnel-row)
Горизонтальная цепочка чипов из `/vacancies/:id/stages`, между чипами серые стрелки `→`.
- Слева отдельный чип «Все (N)» — все, кроме rejected.
- Этапы по порядку (response→offer→hired), `added` системный — НЕ показывается чипом.
- После «Нанят» большой пробел `funnel-gap`, затем отдельный красный «Отказ».
- Чип: цветная точка (цвет этапа) + лейбл + счётчик (моно, мелкий справа).
- Активный чип — заливка `--bg-2` + тёмная рамка.
- Клик по чипу → фильтр по этапу (set локальный `stageFilter`), **закрывает открытую карточку**.

### B.3 Контролы (cand-controls)
- **Поиск** — input с лупой (~280px), ищет по ФИО (query `search`).
- **Bulk-bar** (появляется при выбранных чекбоксах):
  ```
  [N выбрано]  [Перевести]  [Отклонить ▾]  [Сообщение]
  ```
  - «Перевести» → попап выбора этапа → `POST /applications/bulk/move`.
  - «Отклонить ▾» → popover с двумя группами причин (см. B.6) → `POST /applications/bulk/reject`.
  - «Сообщение» → (заглушка/переход к массовой отправке — в MVP можно disabled).
- **Кнопка «Фильтры»** (справа, всегда видна) с badge числа активных фильтров → открывает drawer (B.7).

### B.4 Таблица кандидатов
Данные: `GET /vacancies/:id/applications?stage=&search=&...&sort=&order=` → `ApplicationRow[]` (пагинировано).

Таблица из 2 частей:

**Закреплённая «Профиль» (всегда видна):**
- ☐ чекбокс.
- Avatar (инициалы).
- ФИО + `display_number` (#029) + малая «пипка» цвета этапа.
- Возраст / последний опыт (2 строки серым).
- ScoreBadge `lg` справа.

**Скрол-часть «Остальное» (видна в режиме списка, скрывается при открытой карточке):**
- Телефон + MessIconRound (18px) по `messengers`.
- Зарплата (моно).
- Город.
- Дата отбора (`selected_at`, моно серым).
- StageChip (этап с точкой).

Высота строки ~64px (двустрочный профиль).

**Сортировка:** каждый заголовок кликабелен, стрелки ▲▼. Дефолт `score desc`. Поля: ФИО, AI, телефон, зарплата, город, дата, этап, возраст.

**Индикатор ПдН:** у кандидатов с `has_pdn=true` — зелёный бейдж «✓ ПдН» рядом с ФИО.

### B.5 Поведение строки
- Клик по строке → открывает карточку соискателя справа (detailMode, см. B.8 + ТЗ-6). Навигация на `/vacancies/:id/candidates/:cid`.
- Клик по чекбоксу — multi-select без открытия карточки.
- Активная строка (открыта карточка) — фон `--bg-3` + `box-shadow: inset 4px 0 0 var(--stage-color)`.
- Hover — фон `--bg-3-hover`.

### B.6 Popover «Отклонить»
Две группы причин (из `GET /settings/reject-reasons`):
- **От кандидата** (синий буллет): «Не вышел на связь», «Не устроила ЗП», «Принял другой оффер», «Не устроил график», «Слишком далеко от дома».
- **Со стороны компании** (серый/красный буллет): «Несоответствие опыта», «Несоответствие навыков», «Не прошёл интервью», «Не прошёл СБ», «Завышенные ожидания по ЗП».
Выбор пункта → `POST /applications/{id}/reject` (или bulk) с `reason` + `side`.

### B.7 Drawer фильтров (FilterDrawer)
Открывается справа на полную высоту, затемняющий overlay позади.

**Шапка:** «Фильтры» + кнопка «Сбросить» (если есть активные) + X.
**Закладка:** «🔖 Сохранить настроенный фильтр» (disabled если фильтров нет — в MVP заглушка).

**Секции (collapsible, заголовок + бейдж количества + шеврон):**
| Секция | Контрол | Дефолт |
|---|---|---|
| AI-скоринг | слайдер «от N» (0–100, шаг 5) | 0 |
| Зарплата, тыс ₽ | слайдер «до N» (100–500, шаг 10) | 500 |
| Источник | чипы: HeadHunter / Глафира·Telegram / Авито | — |
| Этап воронки | чипы со всеми этапами + точки | — |
| Город | чипы: Москва / СПб / Новосибирск | — |
| Мессенджер | чипы: Telegram / WhatsApp / Max | — |
| Готовность | чекбокс «Готов к переезду» | off |
| Период отбора | чипы: Сегодня/Неделя/Месяц/Квартал/Всё время | — |
| Повторный отклик | чекбокс | off |

**Подвал:** «Сбросить всё» (secondary, слева) + «Показать N» (primary, справа — N = число кандидатов после фильтрации, можно подтягивать предварительным запросом или считать после применения).

Фильтры маппятся в query-параметры `/vacancies/:id/applications` (ТЗ-2 §5).

### B.8 Режим открытой карточки (detailMode)
Когда кликнули по строке:
- Колонки «остальное» уезжают, карточка соискателя (ТЗ-6) занимает их место.
- Закреплённая «Профиль» остаётся слева → скролл списка + переключение между кандидатами одним кликом.
- Чипы воронки сверху видимы; клик по чипу закрывает карточку + фильтрует.
- Закрытие карточки: X в шапке карточки / клик по чипу / Esc → navigate обратно на `/vacancies/:id`.

Состояние detailMode — в локальном стейте экрана или uiStore (`openCandidateId`).

### B.9 Пустое состояние
Если на этапе никого:
> «На этапе «{label}» пока никого. Кандидаты появятся, как только Глафира продвинет их по воронке.»
