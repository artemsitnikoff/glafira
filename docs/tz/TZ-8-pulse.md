# ТЗ-8. Пульс: главный + карточка сотрудника (Экраны 07 + 08)

> **Кому:** React-агент.
> **Зависит от:** ТЗ-3, ТЗ-2 (`/pulse/*`).
> **Источник UX:** прототипы `PulseMain.jsx`, `PulseDetail.jsx` + описания Экранов 07, 08.
> **Роуты:** `/pulse` (список), `/pulse/:id` (карточка-overlay).

Пост-найм-модуль: работа с уже нанятыми сотрудниками в адаптации. Когда кандидат становится `hired` — он появляется здесь.

---

## ЧАСТЬ A. Пульс — главный (Экран 07, `/pulse`)

### A.1 Шапка
- Заголовок «Пульс».
- Подзаголовок «Адаптация и удержание новых сотрудников».
- Переключатель периода (месяц/квартал/год).

### A.2 KPI-полоса
Данные: `GET /pulse/kpi` (PulseKPI).
4 карточки (кликабельны, ведут в Аналитику→Текучка/Опросы):
- **На адаптации** (`onboarding_count`) — в первые 90 дней.
- **Прошли испытательный** (`passed_probation` + `passed_delta`).
- **Ушли в первые 90 дней** (`left_in_90d` + `left_in_90d_pct`%, красная дельта плохо).
- **eNPS** (`enps`, от −100 до +100).

### A.3 Алерты Глафиры
Данные: `GET /pulse/alerts` (PulseAlertOut[]). Горизонтальный список карточек (если есть).
- Цветная полоска слева по `level`: high красный, mid жёлтый, info синий.
- Иконка + заголовок (`title`) + контекст (`context`).
- Кнопка действия по `action_type`: «Открыть карточку» / «Связаться» / «Запустить опрос».
- X на карточке → `POST /pulse/alerts/:id/dismiss` (скрыть до следующего сигнала).

### A.4 Сегменты (опционально, чипы-пресеты)
`Все` (default) · `На адаптации` (день≤90) · `Прошли испытательный` (день>90) · `🔴 Высокий риск` · `Без опроса >14 дней` · `Уволенные за период`. Маппятся в фильтры запроса.

### A.5 Таблица сотрудников
Данные: `GET /pulse/employees?manager=&department=&risk=&period=&survey_status=&search=&sort=` → EmployeeRow[].

| Колонка | Содержимое |
|---|---|
| ☐ | чекбокс |
| Аватар + ФИО + должность | двустрочная |
| Руководитель | `manager_name` |
| Дата выхода | `start_date` (моно) |
| День адаптации | «День {adapt_day} из {probation_days}» + полоска прогресса (AdaptBar) |
| Риск ухода | плашка low/mid/high (цвета `--risk-*`) |
| Последний опрос | `last_survey_date` + mood (👍/😐/👎 по `last_survey_mood`) |
| Действия | 💬 связаться · 📋 запустить опрос |

**AdaptBar** (полоса адаптации): тонкая, 4 секции (0–30/31–60/61–90/>90), маркер текущей позиции, цвет = цвет риска. Высота 6px, radius 3px, светлый трек + цветная заливка.

**Сортировка/фильтры:** по любой колонке; фильтры руководитель/отдел/период/риск/статус опроса; поиск по ФИО.

### A.6 Поведение
- Клик по строке → `/pulse/:id` (карточка справа).
- «Запустить опрос» → модал (шаблон + получатели).
- «Связаться» → чат.
- Чекбоксы + bulk («Запустить опрос для всех», «Отправить сообщение»).

### A.7 Состояния
- Пусто (нет нанятых): «Здесь появятся ваши сотрудники после первого найма. Закройте первую вакансию — и Пульс начнёт за ними следить.»
- Все прошли испытательный: «🎉 Все ваши сотрудники прошли испытательный. Отличная работа.»

## ЧАСТЬ B. Карточка сотрудника (Экран 08, `/pulse/:id`)

Right-side панель (overlay поверх таблицы), аналог карточки соискателя. Ширина адаптивная, внутренний скролл, шапка липкая.

Данные: `GET /pulse/employees/:id` (EmployeeDetail).

### B.1 Тулбар
| Кнопка | Действие |
|---|---|
| 💬 Связаться | чат во внутреннем мессенджере |
| 📋 Запустить опрос | модал (шаблон + время) |
| Заметка | быстрая внутренняя заметка → `POST /pulse/employees/:id/note` |
| ✕ | закрыть |

### B.2 Шапка
**Левая колонка:**
- Avatar 64px.
- H1: ФИО.
- Подпись: должность · отдел.
- Мета: «Руководитель: {manager_name} · Дата выхода: {start_date} · День {adapt_day} из {probation_days}».

**Правая колонка:**
- Риск-плашка большая цветная: 🟢 Low / 🟡 Mid / 🔴 High (фон `--risk-{x}-soft`, текст/точка `--risk-{x}`).
- Под ней: «Последний опрос: N дней назад · {балл}/5 ⭐».

**Полоса адаптации (AdaptBar большая, высота 12px):**
- 4 секции с подписями фаз сверху.
- Крупный маркер текущей позиции.
- Мини-метки событий: «Welcome-pack ✅», «1-on-1 ✅», «Промежуточная встреча ⏳», «Финальное ревью ⏳».

**Мини-блок «Откуда пришёл»:** источник найма (`hire_source`) · рекрутёр (`recruiter_name`) · срок подбора · ссылка «Открыть исходную карточку соискателя» → `/candidates/:candidate_id`.

### B.3 Алерты Глафиры
Под шапкой, если есть открытые: карточки с цветной полоской + действие («Связаться»/«Спросить»/«Напомнить руководителю»).

### B.4 Табы (5, дефолт «Обзор»)

**6.1 Обзор:**
- AI-сводка от Глафиры (1 абзац).
- 4 мини-плашки: eNPS / Удовлетворённость (`satisfaction_avg`) / Активность (`activity_pct`) / Прогресс плана (`plan_progress_pct`).
- Краткий timeline ключевых событий (последние 5).

**6.2 План адаптации:**
Данные: `GET /pulse/employees/:id/plan` (PlanItemOut[]).
Чек-лист по фазам (welcome/month1/month2/month3). Каждый пункт: чекбокс (→ `PATCH /pulse/plan-items/:id {is_done}`) + заголовок + дедлайн (день N) + ответственный (hr/manager/employee).
Глафира сама генерирует план (бек) — фронт только отображает/отмечает.

**6.3 Опросы:**
Данные: `GET /pulse/employees/:id/surveys` (SurveyOut[]).
Каждый опрос — карточка: дата, тип (weekly/monthly/special/enps), результат (`overall_score`). Раскрытие — вопросы+ответы+комментарий. Сравнение с предыдущим/средним по команде. Кнопка «Запустить новый опрос» → `POST /pulse/employees/:id/surveys {template_key, send_at?}`.

**6.4 Чат:** аналог чата из ТЗ-6, каналы те же (TG/Max/WhatsApp/SMS/встроенный).

**6.5 Действия (timeline):** хронология событий по сотруднику (опрос отправлен/отвечен, комментарий руководителя, алерт, смена статуса).

### B.5 Состояния
- День 1: «{ФИО} начал работать сегодня. Глафира отправит первый welcome-опрос завтра.»
- Прошёл испытательный (status=passed): тулбар + «Закрыть карточку адаптации».
- Уволен (status=left): вместо тулбара баннер «Сотрудник уволен {left_at}» + причина + «Открыть exit-интервью».
