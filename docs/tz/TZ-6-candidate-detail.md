# ТЗ-6. Карточка соискателя (Экран 04)

> **Кому:** React-агент.
> **Зависит от:** ТЗ-3 (компоненты), ТЗ-5 (воронка-хост), ТЗ-2 (candidates, glafira, consents, chat, documents, comments, applications).
> **Источник UX:** прототип `Candidates.jsx` (`CandidateDetail`) + описание Экрана 04.
> **Где монтируется:** right-side панель в воронке (`/vacancies/:id/candidates/:cid`) И на весь экран из общей базы (`/candidates/:id` — частично, основной полноэкранный вид — в ТЗ-7).

Это самый сложный компонент. Делать аккуратно по табам.

---

## 0. Контекст открытия

Карточка открывается в двух режимах:
- **В воронке** (`fromFunnel`): панель справа от списка, ширина ≈60–65% свободной ширины. Контекст — конкретная вакансия (`application_id`).
- **Из общей базы** (`fromPool`): меняет поведение тулбара (см. §4). Полноэкранный вид с историей участия — в ТЗ-7, но сами табы переиспользуются отсюда.

Компонент `CandidateDetail` принимает: `candidateId`, `applicationId?`, `fromPool: boolean`, `onClose`.

---

## 1. Лоадер (cd-loading)

При открытии — оверлей ~850мс:
- 💃 эмодзи (танцующая Глафира).
- Текст «Глафира собирает профиль…» с анимированными точками (`cd-load-dots`).

Это дизайн-приём (имитация работы AI). Реализовать как 850мс таймер ИЛИ показывать пока грузятся `GET /candidates/:id` (что дольше). Не блокировать данные искусственно дольше необходимого.

---

## 2. Тулбар (cd-toolbar)

Действия слева + X справа. Состав зависит от этапа и контекста.

| Кнопка | Тип | Действие |
|---|---|---|
| **Перевести ▾** | success (зелёная) | popover «На какой этап?» — все не-терминальные этапы. Текущий «сейчас», следующий «далее». `fromPool` → попап превращается в выбор вакансии. → `POST /applications/:id/move` |
| **Отклонить ▾** | secondary | popover 2 группы причин (как ТЗ-5 B.6) → `POST /applications/:id/reject` |
| **Комментарий** | secondary | скролл к табу «Комментарии» + фокус поля |
| **ПдН** | secondary (иконка) | видна если согласие не подписано → `POST /candidates/:id/consent/request` |
| **«ПдН ✓»** | зелёная плашка | вместо «ПдН» если `has_pdn`; не кликабельна |
| **✕** | icon-btn | закрыть (onClose) |

### Состояния тулбара
- **Кандидат на финальном `hired`:** «Перевести» заменяется на «Создать сотрудника» (передача в Пульс — но найм уже создаёт employee автоматически на беке; кнопка для явного действия/перехода в карточку сотрудника).
- **Кандидат в `rejected`:** скрыть «Перевести/Отклонить», показать «Восстановить» → `POST /applications/:id/restore`.

### ПдН-бейдж в шапке
У кандидата с `has_pdn` — зелёный бейдж «✓ ПдН» (чуть крупнее, чем в таблице).

## 3. Шапка профиля (cd-header)

Данные: `GET /candidates/:id` (CandidateDetail) + если есть `application_id` — контекст из `GET /applications/:id`.

### 3.1 Контекст-строка (над именем, серая)
- **Source pill** — цветной значок источника:
  - hh → «Отклик с HeadHunter» (красный `--src-hh`),
  - telegram → «Глафира · Telegram» (бирюзовый `--src-tg`),
  - avito → «Авито» (синий `--src-avito`).
- «от {дата отклика}» + название вакансии (если в контексте).

### 3.2 Имя + AI-скоринг
- H1: ФИО (`full_name`).
- Справа ScoreBadge size `xl`.

### 3.3 Опыт-строка
«{опыт} · {last_company} · {last_period}» (напр. «4 года · Сбер · апрель 2024 — наст. время»).

### 3.4 Зарплата
- Большое моно число `{salary_expectation} ₽`.
- Подпись «ожидания».

### 3.5 Теги
- Чипы тегов (`tags`).
- Кнопка «+ Добавить тег» → `POST /candidates/:id/tags`.

### 3.6 Контакт-бокс (правая часть)
Строки-пары «лейбл : значение»:
- **Телефон**: `{phone}` + MessIconRound (22px) по `messengers`.
- **Город**: `{city}`.
- **E-mail**: `{email}`.

---

## 4. Табы (cc-tabs)

7 табов, дефолт **«Резюме»**. Активный — толстое подчёркивание.

| Таб | ID | Источник данных |
|---|---|---|
| Резюме | resume | candidate + evaluation (вердикт) |
| Оценка AI | ai | `GET /candidates/:id/evaluation` |
| Верификация | verify | `GET /candidates/:id/verification` (gated по ПдН) |
| Чат | chat | `GET /candidates/:id/messages` |
| Документы | docs | `GET /candidates/:id/documents` |
| Комментарии | comments | `GET /candidates/:id/comments` |
| Все действия | actions | `GET /home/events?candidate_id=:id` |

## 5. Таб «Резюме» (ResumeTab)

### 5.1 AI-вердикт-карточка (AIVerdictCard) — всегда вверху
```
┌─────────────────────────────────────────────┐
│ 👩🏻  Оценка от Глафиры                  [92] │
│      {summary — краткий вердикт}             │
│  Посмотреть подробную оценку →               │
└─────────────────────────────────────────────┘
```
- Эмодзи 👩🏻, балл (ScoreBadge), `summary` из evaluation.
- Линк → переключает на таб «Оценка AI».
- Если оценки нет — кнопка «Оценить» → `POST /glafira/score`.

### 5.2 Опыт работы
Список `experience`: должность (полужирный) · компания · период (справа серый) · описание.

### 5.3 Навыки
Чипы `skills` (skill-chip). Совпадающие с требованиями — могут подсвечиваться (если данные есть).

### 5.4 Образование
Список `education`: вуз/специальность + годы.

### 5.5 Дополнительно (extra-grid, 2×2 из `extra`)
- Языки · Переезд · Командировки · Удалёнка.
- Под сеткой — **«Предпочтительный способ связи»**: 3 tap-чипа `Telegram` · `Почта` · `Телефон` с иконками (paper-plane / конверт / трубка). Один активный (синий тинт + рамка) = `preferred_channel`. Клик переключает → `PATCH /candidates/:id {preferred_channel}`. Дефолт telegram.

## 6. Таб «Оценка AI» (AITab)

Данные: `EvaluationOut`.
1. Большой балл + лейбл «Подходит/Подходит частично/Не подходит» (по `verdict`).
2. Вердикт — `summary` (1–2 абзаца).
3. Сильные стороны — список `strengths` с 🟢.
4. Слабые/риски — список `risks` с 🟡/🔴.
5. Соответствие требованиям — таблица `requirements_match`:
   | Требование | Статус | Комментарий |
   - status pass → ✅, warn → ⚠️, fail → ❌.
6. Прогноз срока выхода — `forecast`.

Если оценки нет → кнопка «Запустить оценку Глафиры» → `POST /glafira/score`.

---

## 7. Таб «Верификация» (VerifyTab) — ПдН-GATED

**Критично:** доступен только при `has_pdn`. Иначе:
- Замок + «Верификация недоступна» + кнопка «Запросить ПдН» (→ consent request).
- Бек вернёт `403 CONSENT_REQUIRED` если дёрнуть напрямую — фронт обрабатывает.

При наличии ПдН — `GET /candidates/:id/verification` (VerificationOut):

**Мета-строка сверху:** дата проверки + номер согласия (`consent_number` PD-XXX/26) + кнопка «Перепроверить» (→ `POST /candidates/:id/verify`).

**Блоки** (`blocks[]`, 7 шт), все одинаково оформлены:
- иконка-литера слева,
- заголовок (`title`),
- цветные чипсы источников (`sources[]`): тип → цвет (`api`→`--vf-src-api` синий, `reg`→`--vf-src-reg` фиолетовый, `pub`→`--vf-src-pub` оранжевый, `ai`→`--vf-src-ai` красный),
- статус справа (`status`): clean→зелёный «Чисто», info→синий «Инфо», warn→оранжевый «Внимание», risk→красный «Риск».

Блоки по `key`: inn, fssp, bankruptcy, registries, public, ai_intel, alimony. Содержимое `data` рендерить по блоку (ИНН/ФИО/ДР для inn; список производств для fssp; и т.д.).

**AI-разведка (`ai_intel`)** — находки как блоки-цитаты: оранжевая полоса слева 3px, фон `#FFF7F2`, italic, с ссылкой на источник.

---

## 8. Таб «Чат» (ChatTab)

Данные: `GET /candidates/:id/messages` (опц. `?channel=`, `?application_id=`).

### 8.1 Стрим сообщений (chat-stream)
- Day-divider по датам.
- Строка: Avatar (входящие слева / исходящие справа), над пузырём имя отправителя + chip канала (точка цвета канала + лейбл), пузырь, время (моно) под пузырём.
- AI-сообщения (`sender_type='ai'`) — помечены значком 🤖/брендово.
- В `fromPool` — разделители «Контекст: вакансия XYZ» (`application_context`).
- Цвета каналов: TG бирюзовый, hh красный, WhatsApp зелёный, Max синий, SMS серый, E-mail нейтральный.

### 8.2 Compose (chat-compose)
- Селектор канала ответа (dropdown), порядок: Telegram · hh.ru · Max · WhatsApp · SMS · E-mail. У каждого — цветной круг + метка (TG/hh/MX/WA/SMS/@). Дефолт telegram.
- Textarea 2 строки + кнопка отправки (синий paper-plane), disabled пока пусто.
- Подсказка: «Ctrl+Enter — отправить · ответ уйдёт в {канал}».
- Отправка → `POST /candidates/:id/messages {channel, body, application_id?}`.

## 9. Таб «Документы» (DocsTab)

Данные: `GET /candidates/:id/documents`.
Сетка тайлов:
```
┌──────────────────────────┐
│ [PDF]  Резюме.pdf     [↓]│
│        412 КБ · 08.04.26 │
│        импорт hh         │
└──────────────────────────┘
```
- Тип-значок цветной (pdf красный, img зелёный, doc синий).
- Метаданные: размер · дата · источник.
- Кнопка скачать → `GET /documents/:id/download`.
- Drop-tile в конце: «+ Перетащите файл или нажмите» → `POST /candidates/:id/documents` (multipart).

---

## 10. Таб «Комментарии» (CommentsTab)

Данные: `GET /candidates/:id/comments`.
- Лента: Avatar-инициал, шапка (имя · роль · время), текст, @упоминания.
- Compose: textarea 3 строки, подсказка «@ упомянуть · Ctrl+Enter», кнопка «Отправить» (primary) → `POST /candidates/:id/comments {body, mentions}`.

---

## 11. Таб «Все действия» (ActionsTab)

Данные: `GET /home/events?candidate_id=:id` (хронология, свежие сверху).
- Строка: `[icon] [Кто] [роль] [текст события]` + `[время · дата]`.
- AI-события (`actor_type='ai'`) — иконка в розовом круге (брендовый), имя «Глафира» цветом.
- Системные: отклик, переходы по этапам.

## 12. Дизайн / поведение

- Карточка — overlay поверх правой части воронки, ширина адаптивная (≈60–65%).

- Avatar — инициалы, цвет из хеша ФИО.
