# ТЗ-4. Главная / Dashboard (Экран 01)

> **Кому:** React-агент.
> **Зависит от:** ТЗ-3 (каркас, базовые компоненты), ТЗ-2 (`/home/*` эндпойнты).
> **Источник UX:** прототип `Home.jsx` + описание Экрана 01.
> **Роут:** `/home`. Дефолтный экран после входа.

---

## 0. Назначение

Операционный дашборд старшего рекрутёра. Срез «как идут дела»: 6 KPI, что требует внимания, лента событий live, сводка адаптации, топ-источники. Экран смотровой — отсюда проваливаются в вакансии/отчёты, глубоко не работают.

---

## 1. Структура страницы

```
┌──────────────────────────────────────────────────────────────┐
│  Заголовок «Главная» (или «Привет, Анна» по тыку)            │
│  Обновлено 5 мин назад · 14:32      [Неделя·Месяц·Квартал·Год·Всё время] │
├──────────────────────────────────────────────────────────────┤
│  KPI-сетка (6 базовых, +2 расширенных по тыку)                │
├──────────────────────────────────────────────────────────────┤
│  Требуют внимания (≈50%)   │  Лента событий · live (≈50%)    │
├──────────────────────────────────────────────────────────────┤
│  Адаптация · Пульс-Онбординг (сводка + Требуют внимания HR)  │
├──────────────────────────────────────────────────────────────┤
│  Топ-источники кандидатов (опционально)                      │
└──────────────────────────────────────────────────────────────┘
```

---

## 2. Шапка экрана

- Заголовок: «Главная» (дефолт) ИЛИ «Привет, {user.full_name по имени}» по тыку `greeting`.
- Подпись слева: «Обновлено N мин назад · HH:MM» (моно, серый `--fg-3`). Время последнего рефреша.
- Справа — сегментный контрол периода: `Неделя · Месяц · Квартал · Год · Всё время`. Дефолт **Месяц**. Хранится в локальном стейте экрана.
- Смена периода → рефетч `/home/kpi?period=X` и `/home/sources?period=X`. НЕ влияет на «Требуют внимания», «Лента», «Адаптация».

---

## 3. KPI-сетка

Данные: `useQuery(["home","kpi",period])` → `GET /home/kpi?period={period}` → `HomeKpi { period, cards: KpiCard[] }`.

Сетка адаптивная: 3×2 на широком, 2×3 на среднем. 6 базовых карточек:

| key | Метрика | unit |
|---|---|---|
| open_vacancies | Открытые вакансии | — |
| closed_vacancies | Закрытые вакансии | — |
| avg_time_to_hire | Среднее время найма | дней |
| turnover_90d | Текучесть (90 дней) | % |
| active_candidates | Активных кандидатов | — |
| conversion | Конверсия отклик→найм | % |

Расширенный набор (тык `kpiExtended`) добавляет: `cost_per_hire` (₽), `recruiter_response_speed` (часа).

### Карточка KPI (компонент KpiCard)
- Лейбл сверху + мелкая `i`-иконка с тултипом (расшифровка метрики).
- Большое число `--font-mono` ~40px + единица измерения рядом мелкой (если `unit`).
- Низ в две колонки: дельта-чип слева + подпись справа (`caption`).
- **Логика цвета дельты** (`delta_dir` приходит с бека, фронт только красит):
  - `up` / `down-good` → зелёный тинт, символ ▲/▼.
  - `down` / `up-bad` → красный тинт.
  - `flat` → нейтральный серый.
- Карточка некликабельна (смотровое стекло).
- Период «Всё время» → дельты `—` (бек вернёт `delta: null`, `delta_dir: flat`).

## 4. «Требуют внимания» (левая половина, ≈50%)

Данные: `GET /home/attention` → `AttentionItem[]`.

### Шапка блока
- Слева: «Требуют внимания» + пилюля-счётчик (число items).
- Справа: ghost-кнопка «Все вакансии →» → navigate `/vacancies`.

### Строка (компонент AttentionRow)
```
[флажок-иконка]  {vacancy_name}
                 {text}                          [→]
```
- Иконка-флажок по `kind`:
  - `urgent` — красный,
  - `warn` — жёлтый,
  - `deadline` — оранжевый.
- Клик по строке → navigate `/vacancies/{vacancy_id}` (воронка).

---

## 5. «Лента событий · live» (правая половина, ≈50%)

Данные: `GET /home/events?limit=30` → `EventItem[]`. **Polling** `refetchInterval: 15000`.

### Шапка блока
- «Лента событий» + бейдж `live` с пульсирующей точкой (CSS-анимация).

### Строка события (компонент EventRow)
- Круглая иконка 12px слева, цвет по `type`:
  - `qual` синий-бренд, `new` зелёный, `score` золотой, `offer` фиолетовый, `move` серый.
- Текст события: рендер из `text` + `entities`. Имя «Глафира» подсвечено `--brand-accent` (класс `.anatoly`/`.glafira`), сущности (`entities[].type` = candidate/vacancy/client) — линки (класс `.ent`), клик → соответствующий роут.
- Время справа мелким серым (форматировать `created_at` → «5 мин назад», «1 ч назад»).

### Поведение
- Контейнер `maxHeight: 380px`, собственный скролл.
- Группировки по дням нет.

## 6. «Адаптация · Пульс-Онбординг»

Данные: `GET /home/pulse-summary` → структура со сводкой + список HR-внимания. (Бек: расширить `/home/pulse-summary`, отдать `{onboarding_count, onboarding_delta, risk_split:{high,mid,low}, satisfaction_avg, answered_pct, silent_pct, enps, enps_delta, attention_hr: [...] }`.)

### Шапка
- «Адаптация» + светло-серый подзаголовок `· Пульс-Онбординг`.
- Справа ghost «Все сотрудники →» → navigate `/pulse`.

### Сетка статистик (4 карточки в ряд, на <1100px → 2×2)
1. **На адаптации** — крупная цифра + подпись `+N в этом месяце`.
2. **Риск ухода** — горизонтальный stacked-бар (красный·жёлтый·зелёный пропорционально), под ним легенда с числами.
3. **Средняя оценка** — «N.N / 5» + подпись `ответили X% · Y молчат`.
4. **eNPS** — цифра (стаж ≥90 дн) + дельта-чип + подпись `к прошлому периоду`, мелкое `90 дн.` у заголовка.

### «Требуют внимания HR» (под разделителем)
- Лейбл `ТРЕБУЮТ ВНИМАНИЯ HR` (uppercase, мелкий) + красная пилюля-счётчик.
- Строки (тот же паттерн, что §4): цветная иконка-флажок (`urgent`/`warn`), имя + роль (роль приглушённым), причина (короткая строка), справа моно `день N` + risk-pill `риск NN` (красный/жёлтый/зелёный, пороги 70/50/0), стрелка `›`.
- Клик → navigate `/pulse/{employee_id}` (Экран 08).

Блок всегда виден (не за тыком).

## 7. «Топ-источники кандидатов» (опционально, тык `showSources`, дефолт ON)

Данные: `GET /home/sources?period={period}`.

Горизонтальные бары: hh.ru, Авито Работа, Telegram-бот Глафиры, Рефералы, Прямые отклики. Цвета — бренды источников (`--src-hh`, `--src-avito`, `--src-tg`). Длина бара пропорциональна значению, число справа моно. В шапке блока — текущий период мелким серым.

---

## 8. Тыки (Tweaks)

Локальный стейт экрана (или uiStore). В MVP — можно зашить значения по умолчанию + простой dev-переключатель:
| Ключ | Эффект | Дефолт |
|---|---|---|
| `greeting` | «Привет, Анна» вместо «Главная» | off |
| `kpiExtended` | +CPH +Скорость отклика | off |
| `showSources` | Показывать топ-источники | on |
