# ТЗ-10. Настройки (Экран 09)

> **Кому:** React-агент.
> **Зависит от:** ТЗ-3, ТЗ-2 (`/settings/*`, `/users`).
> **Источник UX:** прототип `Settings.jsx` + описание Экрана 09.
> **Роут:** `/settings?tab={key}`.

Конфигурация: профиль, команда, интеграции, поведение Глафиры, шаблоны, биллинг.

---

## 1. Структура

**В реализации** (по описанию) подменю — **верхняя горизонтальная панель табов** (`SettingsTopTabs`) для разделов первого уровня: Профиль · Команда · Интеграции · Глафира · Шаблоны · Воронка по умолчанию · Биллинг · Прочее. Внутри раздела — контент-флоу с карточками.

Активный таб — из `?tab=`. Контент — макс. ширина ~880px, центрирован. Между карточками 24px.

---

## 2. Типовые паттерны контролов

- Inputs (text/email/phone/password с глазиком).
- Toggle (on/off).
- Slider (числовые пороги).
- Chips (мульти-выбор).
- Select/Combobox.
- File upload (аватар/логотип).
- Textarea/Rich-text (шаблоны с подсветкой переменных `{{var}}`).

Общая сетка страницы: Заголовок + [действие Сохранить/Импорт] → Подзаголовок → карточки с группами полей.

Поведение: изменения подсвечиваются жёлтой точкой у поля + баннер «Не забудьте сохранить». Кнопка «Сохранить» sticky в подвале если страница длинная. «Отменить» сбрасывает к последнему сохранённому.

## 3. Разделы (детально)

### 3.1 Профиль (`?tab=profile`)
Данные: `GET/PATCH /settings/profile`.
- Аватар (drag-n-drop / кнопка).
- ФИО, email, телефон, должность.
- Часовой пояс (combobox).
- Смена пароля (отдельная карточка с подтверждением).
- «Сохранить изменения».

### 3.2 Команда (`?tab=team`)
Данные: `GET /users`, `POST /users`, `PATCH /users/:id`.
- Таблица: avatar + ФИО + email + роль + статус + действия (Редактировать/Деактивировать).
- «+ Пригласить» (модал: email + роль + опц. вакансии).
- Фильтр по роли и статусу.

### 3.3 Интеграции (`?tab=integrations`)
Данные: `GET/PATCH /settings/integrations`.
Каждый источник — карточка: лого + название + статус (зелёная точка «Подключено» / серая «Не подключено») + «Настроить»/«Отключить» + параметры в раскрытом виде.
Группы:
- Источники найма: hh.ru (ключ API, регионы, шаблоны), Авито (токен, лимиты), Telegram (бот), Реф-программа.
- Мессенджеры: WhatsApp Business, Max, SMS.
- Календари: Google/Outlook/Яндекс.
- Корп-системы: 1С/SAP HR.
> В MVP реально настраиваются hh и Авито (остальные — UI-заглушки со статусом «Не подключено»).

### 3.4 Глафира (`?tab=glafira`)
Данные: `GET/PATCH /settings/glafira` (GlafiraSettings).
- **Голос и тон** — 3 карточки-радио (Дружеский / Формальный / Деловой) с превью реплик. → `tone`.
- Доп.: «на ты/вы» (`use_informal`), «Эмодзи: умеренно/нет» (`emoji_level`), «Подпись от рекрутёра».
- **Скрипты скрининга** — глобальные шаблоны вопросов + дефолт.
- **Правила автодействий** — «if-then» карточки:
  ```
  Если: AI-скоринг < {auto_reject_below}  → Отклонить автоматически
  Если: {days_no_response} дней нет ответа → Закрыть «Не вышел на связь»
  ```
  Слайдеры порогов в шапке (`auto_reject_below`, `auto_select_above`, `days_no_response`).
- **Стоп-слова/стоп-факторы** — глобальный блок-лист (chips).
- **Режим по умолчанию** — A/B/C (`default_mode`).

### 3.5 Воронка по умолчанию / Причины отказа (`?tab=funnel`)
Данные: `GET/POST/DELETE /settings/reject-reasons`.
Глобальный справочник причин (используется в Экранах 03, 04, 06). Две группы, каждая — ряд чипсов-пилюль с точкой-индикатором слева + крестик справа:
- **От кандидата** (синий буллет): «Не вышел на связь», «Не устроила ЗП», «Принял другой оффер», «Не устроил график», «Слишком далеко от дома».
- **Со стороны компании** (красный/серый буллет): «Несоответствие опыта», «Несоответствие навыков», «Не прошёл интервью», «Не прошёл СБ», «Завышенные ожидания по ЗП».
В конце каждой группы — пунктирная чипса «+ Добавить» (→ POST). Удаление — крестик (→ DELETE).

### 3.6 Шаблоны (`?tab=templates`)
Данные: `GET/POST/PATCH /settings/email-templates`, `/settings/survey-templates`.
- **Письма** — список (название, тип события, ENABLED toggle). Редактор: тема + тело + переменные (`{{candidate_name}}`, `{{vacancy}}`, `{{recruiter_name}}`, `{{interview_link}}`, `{{salary}}`), превью с подстановкой, «Тестовая отправка на мою почту».
- **Сообщения в чат** — короткие шаблоны для TG/WhatsApp.
- **Опросы Пульса** — список («Welcome день 7», «Адаптация день 30», «eNPS квартальный»). Каждый: вопросы с типами (1–5/1–10/текст/multi), условие отправки (день/каждые N дней), каналы (email/TG/встроенный).

### 3.7 Биллинг (`?tab=billing`) — заглушка в MVP
Данные: `GET /settings/billing`.
- Карточка тарифа: план, цена/мес, дата списания.
- Лимиты с прогресс-барами: «Вакансий 12/25», «AI-токенов 480к/1М».
- «Изменить тариф» (модал-сравнение).
- История платежей (таблица).

### 3.8 Прочее (`?tab=other`)
- Общие: язык, часовой пояс, формат даты.
- API и вебхуки.
- Логи и аудит (для админов) — читает `audit_log`.

---

## 4. Состояния
- Загрузка → скелетоны на полях.
- Сохранение → спиннер на кнопке.
- Ошибка → красная полоска под полем + текст.
- Успех → toast «Сохранено».
- Нет прав → баннер «У вас нет прав на редактирование. Обратитесь к админу команды.»
