# ТЗ-7. Общая база кандидатов (Экран 05)

> **Кому:** React-агент.
> **Зависит от:** ТЗ-3, ТЗ-6 (табы карточки переиспользуются), ТЗ-2 (`/candidates/*`).
> **Источник UX:** описание Экрана 05.
> **Роуты:** `/candidates` (сетка), `/candidates/:id` (карточка на весь экран).

Общая база всех кандидатов компании, независимо от вакансий. Двухколоночный layout (сайдбар + контент на всю ширину), без субменю.

---

## A. Список (4A, `/candidates`)

Данные: `GET /candidates?search=&city=&exp=&score_min=&score_max=&source=&vacancy_id=&stage=&tags=&added_period=&sort=&order=&page=` → `Paginated<CandidateGridItem>`.

### A.1 Шапка раздела (sticky)
- Заголовок «Кандидаты».
- Подзаголовок-счётчик: «{total} кандидатов в базе» (или «Показано {items} из {total}» при фильтрах).
- Справа: «+ Добавить кандидата» · «Импорт из файла» (импорт — в MVP заглушка/базовый).

### A.2 Панель управления (sticky)
- **Слева:** поиск по ФИО (лупа, дебаунс 200мс). Опц. переключатель «расширенный поиск».
- **Справа:** Сортировка (dropdown): По дате добавления (новые) · По AI-скорингу · По ФИО А-Я · По дате активности → query `sort`/`order`.

### A.3 Строка фильтров
| Фильтр | Тип | query |
|---|---|---|
| Город | dropdown с поиском | `city` |
| Опыт | сегментный (Все/до 1/1–3/3–5/5+) | `exp` |
| AI-скоринг | сегментный (Все/🟢80+/🟡50–79/🔴0–49) | `score_min`,`score_max` |
| Источник | dropdown | `source` |
| Вакансия | dropdown с поиском | `vacancy_id` |
| Этап | dropdown (все этапы + «В базе») | `stage` |
| Теги | мульти-селект | `tags` |
| Дата добавления | сегментный (Неделя/Месяц/Квартал/Год/Всё время) | `added_period` |

Справа — «Сбросить» (видна если что-то применено). **Все фильтры синхронизируются с URL** (query params, ссылку можно шарить) — использовать `useSearchParams`.

### A.4 Сетка карточек
`grid auto-fill, minmax(~280px, 1fr), gap 16px`. 4 в ряд на широких, 2–3 средних, 1 узких.

**Карточка кандидата (CandidateGridItem):**
```
┌──────────────────────────────┐
│ [Avatar]  {full_name}    [87]│  ← ScoreBadge цветной справа
│                              │
│ {age} лет                    │
│ {last_position}              │
│ {last_company} · {last_period}│
│ ──────────────────────────   │
│ ▣ {last_vacancy.name}        │
│   {StageChip}          [+N]  │  ← other_vacancies_count
└──────────────────────────────┘
```
- Шапка: Avatar + ФИО (bold, эллипсис) + ScoreBadge справа (null → серый «—»).
- Инфа: возраст / должность / компания·период (мелко, плотно).
- Разделитель.
- Лейбл вакансии (внизу): маркер + название `last_vacancy` + StageChip этапа. Если `other_vacancies_count > 0` — бейдж «+N» (hover → список всех вакансий). Если `last_vacancy = null` — «🔘 В базе · не привязан к вакансии».
- Дубль (`is_duplicate`) — мелкий бейдж «Дубль» в углу.
- ПдН (`has_pdn`) — можно показать зелёную точку/бейдж.

**Hover:** лёгкое поднятие (тень) + иконка ⋯ (Открыть · Назначить на вакансию · Написать · Объединить дубли · Удалить).

### A.5 Поведение
- Клик по карточке → `/candidates/:id` (4B).
- Клик по ScoreBadge → карточка с активным табом «Оценка AI» (`?tab=ai`).
- Клик по лейблу вакансии → `/vacancies/:vacancy_id`.
- Клик по «+N» → карточка с раскрытым блоком «История участия».
- Бесконечный скролл по 24 + скелетон внизу (`useInfiniteQuery`).

### A.6 Edge cases
- Нет кандидатов → empty: иллюстрация + «В базе пока нет кандидатов» + CTA (Добавить вручную / Импортировать / Подключить Анатолия).
- Фильтры пусто → «Никого не найдено · Сбросить фильтры».
- Loading → 8 карточек-скелетонов.
- Длинное ФИО/вакансия → эллипсис + тултип.
- Нет скоринга → серый «—».

## B. Карточка на весь экран (4B, `/candidates/:id`)

Сайдбар виден, остальная область — карточка на всю ширину. Переиспользует табы из ТЗ-6 (`fromPool=true`).

### B.1 Шапка кандидата (sticky)
- **«← Назад к кандидатам»** вверху — возврат в список с сохранёнными фильтрами и позицией скролла.
- Строка 1 — имя (bold) + действия справа: «Написать» · «Позвонить» · «Назначить на вакансию» · ⋯ (Скачать резюме, Объединить дубли, Удалить, Добавить тег).
- Строка 2 — инфа одной строкой через «/»: `{age} лет / {last_position} / {last_company} / {last_period}`.
- Строка 3 — контакты: телефон (кликабельный) + бейджи мессенджеров (клик → таб Чат с каналом) + email.
- Строка 4 — город: «Город: {city} ({region})».
- Строка 5 — теги (чипы) + кнопка «+».
- AI-скоринг — крупный ScoreBadge справа, клик → таб «Оценка AI».

### B.2 Блок «История участия в вакансиях»
Под шапкой, перед табами. Главное отличие от карточки в воронке.
Данные: `GET /candidates/:id/applications` → `ApplicationHistoryItem[]`.

Заголовок: «История участия в вакансиях ({N})».
Каждая строка:
```
▣ {vacancy_name}              {StageChip}
  Заказчик: {client_name} · {vacancy_status лейбл}
  Отбор: {selected_at} · Скоринг: {ScoreBadge}
  [если rejected] → Отказ: {stage_changed_at} · Причина: {reject_reason}
```
- Текущий/финальный этап с цветовой индикацией.
- Заказчик + рекрутёр (мелким серым).
- Дата отбора; для терминальных — дата перехода.
- Скоринг на момент отбора.
- На hover строки — стрелка `→`, клик → `/vacancies/:vacancy_id/candidates/:candidateId` (карточка внутри вакансии).

Сортировка: по дате отбора, новые сверху (активные можно первыми).
Если ни в одной вакансии — плашка «Кандидат пока ни в одной вакансии · Назначить на вакансию».

### B.3 Вкладки
Те же 7 табов из ТЗ-6, дефолт «Резюме». В `fromPool`:
- **Чат** — полная история по всем вакансиям с разделителями «Контекст: вакансия XYZ».
- **Оценка AI** — отдельные блоки оценки по каждой вакансии.
Активный таб — из `?tab=` query.

### B.4 Поведение
- «← Назад» сохраняет фильтры/скролл (хранить состояние списка в Zustand или восстанавливать из URL).
- Клик по строке истории → карточка внутри вакансии.
- «Назначить на вакансию» → модалка (поиск вакансий + выбор этапа) — в MVP базовая.
- Клик по мессенджеру → таб Чат с каналом.

### B.5 Edge cases
- Нет резюме → структурированные ответы скрининга + «Запросить резюме».
- Дубль → бейдж + модалка «Объединить».
- Анонимизирован/удалён по 152-ФЗ → read-only + причина.
- Loading → скелетон шапки + истории + табов.
