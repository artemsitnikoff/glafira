# Ветка GOV. Кадровый резерв

> Адаптация продукта под государственного заказчика (пилот — Минцифры Новосибирской области). **Минимальные изменения**: переименовываем раздел «Кандидаты» в «Кадровый резерв» и насыщаем его гос-процессами. Всё остальное живёт как в коммерческой версии.

## 1. Принцип

**Что НЕ трогаем (вообще):**
- Раздел «Вакансии» (Экраны 02, 03) — название, поля, процесс, источники, воронка. Всё работает как сейчас.
- Карточка соискателя (Экран 04) — структура, табы, верификация.
- Главная, Аналитика, Пульс, Настройки, Сайдбар.
- Источники откликов (hh, Авито, Telegram-Глафира) — остаются.
- Логика воронки и этапы — остаются.

**Что меняем:**
- Раздел **«Кандидаты»** (Экран 05) → **«Кадровый резерв»**. Внутри — гос-карточка резервиста + гос-фильтры.
- В **карточке вакансии** (Экран 03, шапка) — добавляем **одну кнопку**: «Подобрать из кадрового резерва».

Всё.

## 2. Раздел «Кадровый резерв» (бывшие «Кандидаты»)

### 2.1 Изменения в сайдбаре

В `Sidebar.jsx` пункт «Кандидаты» переименовывается в **«Кадровый резерв»** (иконка та же — `users`). Маршрут `/candidates` остаётся (для обратной совместимости в коде) либо переименовывается в `/reserve` — на усмотрение разработки.

### 2.2 Список (Экран 05, режим списка)

Сохраняется существующий лейаут: сетка карточек, фильтры, сортировка. Меняется **состав фильтров и полей на карточке**.

**Заголовок:** «Кадровый резерв».
**Подзаголовок-счётчик:** «247 резервистов в базе» (вместо «1 247 кандидатов»).

#### Фильтры (заменяют коммерческие)

| Фильтр | Значения |
|---|---|
| **Группа должностей** | Все · Высшая · Главная · Ведущая · Старшая · Младшая |
| **Категория** | Все · Руководители · Помощники (советники) · Специалисты · Обеспечивающие специалисты |
| **Гос. орган-источник** | Все / конкретный (выпадающий список) |
| **Допуск к гос. тайне** | Все · Без допуска · 3 форма · 2 форма · 1 форма |
| **Стаж гос. службы** | Все · до 1 года · 1–3 · 3–5 · 5+ |
| **Срок в резерве** | Все · истекает в 3 мес · в 6 мес · в год · истёк |
| **Декларация о доходах** | Все · Актуальна · Просрочена · Не требуется |
| **Статус в резерве** | Все · Активный · Приостановлен · Исключён |
| **Город** | как было |
| **Дата включения в резерв** | Неделя · Месяц · Квартал · Год · Всё время |

#### Карточка в списке

Сохраняется визуально похожая на коммерческую, но содержит гос-поля:

```
┌─────────────────────────────────┐
│ [Аватар]   Петренко И.С.    🟢 │
│            Группа: Ведущая      │
│                                 │
│ 45 лет · Стаж г/с 8 лет         │
│ Допуск: 2 форма                 │
│ В резерве: ведущая, до 03.27    │
│                                 │
│ ─────────────────────────────   │
│ 🏛 Источник: Минфин НСО          │
│ ✓ Декларация · ✓ Антикор        │
└─────────────────────────────────┘
```

Цветной бейдж справа от ФИО — **индекс готовности** (вместо AI-скоринга):
- 🟢 Готов к назначению (документы свежие, нет рисков)
- 🟡 Нужна актуализация (что-то просрочено)
- 🔴 Есть риски (антикор-флаги, истекает срок резерва)

Внизу карточки — мини-статусы документов: декларация, антикор-проверка, допуск.

### 2.3 Карточка резервиста (Экран 05, режим открытой карточки)

Структура — как у соискателя в коммерческой версии (шапка + табы). Меняется содержимое.

#### Шапка
- ФИО, возраст
- Группа должностей + категория (большим)
- Срок пребывания в резерве: «до 03.2027 (осталось 10 месяцев)»
- Гос. орган-источник
- Допуск к гос. тайне + дата оформления
- № приказа о включении в резерв
- Контакты (телефон, e-mail, мессенджеры — как было)
- Бейдж индекса готовности

#### Табы

Те же, что в коммерческой карточке, плюс гос-специфика **внутри** существующих табов:

| Таб | Что внутри (gov) |
|---|---|
| **Резюме** | → переименовать в **«Анкета»**. Структурированные блоки: основное, образование, опыт гос. службы (отдельно от коммерческого), классные чины, доп. проф. образование, языки, сертификаты |
| **Оценка AI** | → переименовать в **«Соответствие требованиям»**. Показывает: на какие группы должностей формально подходит, сравнение с эталоном квалификационных требований по группе |
| **Верификация** | Существующий модуль из коммерческой версии (Экран 04 п. 8.5) переиспользуется **как есть**. Просто становится обязательным (а не опциональным «после ПдН») |
| **Чат** | Как было — TG, e-mail, WhatsApp |
| **Документы** | Добавляются типы: декларация о доходах (с датой подачи), справка о судимости, диплом, удостоверение о повышении квалификации, заявление в резерв, согласие на ПДн |
| **Комментарии** | Без изменений |
| **Все действия** | Без изменений — лента событий по резервисту |

> **Важно:** Добавляется один новый таб — **«История назначений»**. Хронология: на какие вакансии его подбирали, какой был результат (назначен / отказался / не выбран), даты, кто решал. Это критично для гос. контекста — резервист может пройти несколько процедур назначения за время пребывания в резерве.

#### Новые блоки в шапке/анкете

В рамках существующей структуры карточки добавляются гос-поля:

**Блок «Допуск и статусы»** (в шапке)
- Статус в резерве, дата включения, № приказа
- Срок пребывания + счётчик
- Группа и категория должностей
- Допуск к гос. тайне

**Блок «Опыт гос. службы»** (в табе Анкета, отдельно от обычного опыта)
- Места работы с уровнем (фед/рег/мун)
- Классные чины
- Стаж гос. службы (автосчётный)

**Блок «Антикоррупционная проверка»** (в табе Верификация — там уже есть похожее)
- Декларация о доходах (статус, дата)
- Проверка по реестру дисквалифицированных
- Конфликт интересов (через ЕГРЮЛ — уже есть в модуле верификации)
- Реестр коррупционных правонарушений

## 3. Кнопка «Подобрать из кадрового резерва» в вакансии

Это **единственное изменение в разделе «Вакансии»** (Экран 03). Кнопка живёт в шапке воронки рядом с существующими «Поделиться», «Перейти на hh», «Редактировать», «+ Кандидат».

### 3.1 UI

```
┌──────────────────────────────────────────────────────────────┐
│  Frontend-разработчик (Senior)                               │
│  Минцифра НСО · ведёт А. Седова · Новосибирск · от 15.03.26  │
│                                                              │
│  [Поделиться] [Перейти на hh] [Редактировать]                │
│  [🏛 Подобрать из кадрового резерва] [+ Кандидат]            │
└──────────────────────────────────────────────────────────────┘
```

Кнопка с иконкой 🏛 (или похожей гос-семантикой), цвет — нейтральный secondary.

### 3.2 Что происходит при клике

Открывается **модал «Подбор из кадрового резерва»** поверх воронки.

#### Шаг 1. Глафира матчит
Сразу при открытии модала Глафира за несколько секунд проходит по всему резерву и подбирает релевантных. Лоадер с фирменным «Глафира подбирает из резерва…».

Критерии матчинга (Глафира делает их сама на основе данных вакансии и анкет резервистов):
- Соответствие группе должностей (если вакансия — гос. должность)
- Соответствие квалификационным требованиям (стаж, образование)
- Профильный опыт работы
- Допуск к гос. тайне (если нужен по вакансии)
- Свежесть документов (декларация, повышение квалификации)
- Отсутствие антикоррупционных флагов

#### Шаг 2. Список лучших

Модал показывает топ-10 резервистов с обоснованием:

```
┌─────────────────────────────────────────────────────────────┐
│  Подбор из кадрового резерва · 10 подходящих                │
│  Вакансия: Frontend-разработчик (Senior) · Минцифра НСО     │
├─────────────────────────────────────────────────────────────┤
│  ☐ Петренко И.С.                              🟢 92         │
│    Группа: Ведущая · Стаж г/с: 8 лет                        │
│    ✓ Допуск 2ф · ✓ Декларация свежая · ✓ Антикор чисто     │
│    Почему подходит: профильный опыт в Минфине НСО (4 года), │
│    стек совпадает на 87%, декларация подана 2 мес назад.    │
│                                                             │
│  ☐ Корнеев А.В.                               🟢 87         │
│    ...                                                      │
│                                                             │
│  ☐ Сидоров П.М.                               🟡 71         │
│    ...                                                      │
│    Внимание: декларация просрочена 3 мес. Нужна             │
│    актуализация перед назначением.                          │
└─────────────────────────────────────────────────────────────┘
│                          [Отмена] [Добавить выбранных →]    │
└─────────────────────────────────────────────────────────────┘
```

Каждая строка:
- Чекбокс
- ФИО + индекс соответствия (балл 0–100)
- Группа / стаж
- Мини-чек: допуск, декларация, антикор
- **Объяснение Глафиры**: почему подходит именно этот человек (2–3 строки на естественном языке)
- Если есть риски — отдельная жёлтая/красная строка с пометкой

Над списком — мини-фильтры (сужать выдачу):
- Только с готовыми документами
- Только из вашего ведомства
- С допуском не ниже [N]

#### Шаг 3. Добавление в воронку

Пользователь отмечает чекбоксами нужных и жмёт **«Добавить выбранных»**. Резервисты появляются в воронке вакансии на этапе **«Отобран»** (тот же этап `selected` из существующих `STAGES`). 

В карточке такого кандидата (внутри воронки) появляется индикатор-источник: вместо «Отклик с HeadHunter» — «🏛 Из кадрового резерва». Логика чата, переписки, переводов по этапам — **полностью та же**, что для откликов hh.

### 3.3 Что Глафира делает после добавления

Опционально, по согласию пользователя (чекбокс в модале):
- **Уведомить резервистов**: «Вам доступна вакансия [X]. Готовы рассмотреть?» — через предпочтительный канал.
- **Запросить актуализацию документов** у тех, у кого что-то просрочено.

По умолчанию — выключено: рекрутёр (кадровик) сам решает, когда писать.

### 3.4 Что показывается в Карточке резервиста после

В табе «История назначений» появляется запись:
- Вакансия (название, гос. орган)
- Дата подбора
- Статус: «В работе», «Согласился», «Отказался», «Назначен», «Не выбран»

Это естественным образом отражается, поскольку резервист уже в воронке вакансии — система просто проецирует существующие этапы воронки в историю карточки резервиста.

## 8. Чего НЕТ в этой ветке (осознанно)

Чтобы не разрастать пилот:

- Полноценный процесс открытого конкурса (публикация на gossluzhba.gov.ru, тестирование, протоколы комиссий) — НЕ делаем. Только подбор из резерва. Конкурс — это следующая ветка.
- Локальный LLM (GigaChat / YandexGPT) — НЕ делаем для MVP. Пилот идёт на Claude. Замена LLM — отдельный пакет работ, нужен только перед промышленным внедрением в защищённый контур.
- Интеграция со СМЭВ и ЕСИА — НЕ делаем для MVP. Авторизация — стандартная, реестры — через те же источники, что в коммерческой версии.
- Электронная подпись приказов — НЕ делаем для MVP. Глафира готовит проекты документов в PDF, подписание — вне системы.
- Аттестация по ФСТЭК — НЕ делаем для MVP. Делаем перед промышленным внедрением.

Это значит: **MVP можно запустить как обычное облачное решение** (с подписанием NDA и стандартными мерами безопасности коммерческой версии), а под промышленное внедрение позже — отдельный контур.
