# 12 — Импорт кандидатов

> Открывается по кнопке **«Импорт кандидатов»** в шапке экрана **Кандидаты (общая база)** (Экран 05), рядом с «Добавить кандидата».
> Пошаговый визард импорта кандидатов в **общую базу** Глафиры. На первом шаге — **развилка источника**: из файла (Excel) или из **Потока** (по API-токену).
> Десктоп-first. Прототип: `Глафира Рекрутер.html` → раздел «Кандидаты» → «Импорт кандидатов».

## Зачем это нужно

Рекрутёр хочет залить кандидатов в Глафиру из внешнего источника. Источников два:

- **Файл** — Excel-выгрузка из любой системы (hh, Хантфлоу, ручная таблица). Колонки везде разные, поэтому Глафира распознаёт их и даёт сопоставить с полями кандидата.
- **Поток** — прямое подключение по API-токену. Глафира сама забирает кандидатов и резюме, маппинг колонок не нужен.

Во всех случаях кандидаты заливаются **в общую базу — без привязки к вакансии и этапу воронки**. Дальше с ними работают как с обычными кандидатами базы.

- Объёмы могут быть большими (тысячи строк/записей) — прогресс и превью обязательны.
- Импорт честен: то, что недоступно вне исходной системы (файл резюме), мы не обещаем подтянуть.

## Точка входа

| | |
|---|---|
| **Откуда** | Кнопка `Импорт кандидатов` (вторичная, иконка загрузки) в шапке экрана «Кандидаты». |
| **Тип UI** | Полноэкранный пошаговый визард в рабочей области (как «Новый кандидат» / «Новая вакансия»). |
| **Закрытие** | Крестик в шапке визарда → возврат на список базы. |
| **Навигация** | Можно вернуться на шаг назад. Индикатор шагов закреплён сверху и **зависит от выбранного источника**. |

---

## Общая модель — развилка источника

Первый шаг всегда **«Источник»**. Дальше поток разный:

```
                          ┌─ ② Загрузка → ③ Колонки → ④ Превью → ⑤ Готово     (Файл)
① Источник ──────────────┤
                          └─ ② Токен Потока ────────→ ③ Превью → ④ Готово       (Поток)
```

Индикатор шагов в шапке визарда подстраивается под источник:

| Источник | Шаги индикатора |
|---|---|
| **Файл** | Источник · Загрузка · Колонки · Превью · Готово |
| **Поток** | Источник · Токен Потока · Превью · Готово |

Текущий шаг подсвечен, пройденные — с галочкой.

---

## Шаг 1 — Источник (развилка)

Заголовок «Откуда импортировать кандидатов?» + плашка от Глафиры (💃): «Выберите источник — Глафира заберёт кандидатов в общую базу».

Две активные карточки выбора + две заглушки «Скоро» (равные по весу):

| Карточка | Иконка | Описание | Теги | Действие |
|---|---|---|---|---|
| **Импорт из файла** | загрузка (синяя) | Excel-выгрузка из любой системы — hh, Хантфлоу, таблица. Сопоставите колонки вручную. | `.xlsx` `.xls` `маппинг` | «Выбрать файл →» |
| **Импорт из Потока** | 💃 (персиковая) | Подключение по API-токену. Глафира сама заберёт кандидатов и резюме — без файла и маппинга. | `API-токен` `резюме` `без маппинга` | «Подключить Поток →» |
| **Импорт из Talantix** `Скоро` | «T» (синяя) | Подключение по API-токену — как Поток. | `API-токен` `резюме` | заглушка, **не нажимается** |
| **Импорт из Хантфлоу** `Скоро` | портфель (зелёная) | Подключение по API-токену — как Поток. | `API-токен` `резюме` | заглушка, **не нажимается** |

Клик по активной карточке → соответствующая ветка (`Загрузка` или `Токен Потока`). Карточки **Talantix** и **Хантфлоу** — это такие же системы, как Поток (подключение по токену), но интеграции в разработке: помечены бейджем **«Скоро»**, визуально приглушены (пунктирная рамка) и **не кликабельны**.

## Ветка «Файл»

### Шаг 2 — Загрузка файла

**Состав:** drag-and-drop зона + кнопка «Выбрать файл», иконка загрузки. Принимает `.xlsx`, `.xls`. Подсказка про поддержку выгрузок из разных систем.

**Состояния:**

| Состояние | Поведение |
|---|---|
| `idle` | Пустая drop-зона + кнопка выбора. Подсветка зоны при перетаскивании файла. |
| `parsing` | Лоадер «Глафира читает файл…» (персонаж 💃). |
| `done` | Карточка-итог: имя файла, «Найдено N строк · M колонок», распознанные колонки чипами, кнопка замены файла. |
| `error` | «Не удалось прочитать файл. Поддерживаются только .xlsx и .xls». Кнопка «Выбрать другой файл». |

Кнопка «Назад» возвращает на выбор источника. > **Edge case.** Файл не Excel / повреждён → состояние `error`, переход дальше заблокирован.

### Шаг 3 — Сопоставление колонок (главный экран ветки «Файл»)

Заголовок «Сопоставьте колонки» + плашка Глафиры: «Глафира распознала колонки автоматически — проверьте и поправьте, если нужно».

**Таблица сопоставления** — по строке на каждую колонку файла:
- **Слева** — название колонки из файла + 2–3 примера значений мелким серым.
- **Стрелка →**
- **Справа** — дропдаун «Поле кандидата».

Правила:
- Авто-распознанные — предзаполнены, помечены зелёным ✓ «распознано».
- Нераспознанные — подсвечены «выберите вручную», по умолчанию «Не импортировать».
- Одно наше поле — на одну колонку: занятые поля помечаются «занято».

**Индикатор обязательных полей** над таблицей: **Имя ✓** · **Контакт (телефон / email) ✓**.

> **Валидация.** Кнопка «Далее → Превью» **заблокирована**, пока не замаплены **Имя** и хотя бы один контакт. Подсказка: «Сопоставьте Имя и хотя бы один контакт».

**Поля кандидата:** Имя\*, Телефон, Email, Город, Возраст, Зарплата, Источник, Должность, Компания, Опыт, Комментарий, Резюме-ссылка, + «Не импортировать».

> **Решение по продукту.** Этап воронки исходной системы **не импортируется** — кандидаты заливаются в общую базу без привязки к вакансии. Поля «Этап» в маппинге нет.

## Ветка «Поток»

### Шаг 2 — Токен Потока

Заголовок «Подключение к Потоку» + плашка Глафиры: «Вставьте API-токен из Потока — Глафира подключится и сразу покажет превью кандидатов».

**Карточка ввода токена:**
- Поле «API-токен Потока» с иконкой ключа, моноширинный плейсхолдер `pk_live_••••…`.
- Кнопка **«Вставить демо-токен»** (для демо/ревью подставляет валидный токен).
- Подсказка: «Токен можно создать в Потоке: **Настройки → API → Создать токен**. Достаточно доступа на чтение кандидатов».

**Мини-объяснение из 3 шагов:** 1) Глафира подключится к аккаунту Потока · 2) Заберёт кандидатов и их резюме · 3) Покажет превью перед заливкой в базу.

**Состояния:**

| Состояние | Поведение |
|---|---|
| `idle` | Форма ввода токена. Кнопка «Подключиться и загрузить →» заблокирована, пока поле пустое. |
| `connecting` | Лоадер «Глафира подключается к Потоку…» (персонаж 💃), подпись «Забираем кандидатов и резюме по API». |

Кнопка «Назад» возвращает на выбор источника.

> **Ключевое отличие.** После подключения Глафира идёт **сразу на Превью** — шаг сопоставления колонок **пропускается** (поля приходят из API уже структурированными).

> **Валидация.** Пустой токен → primary-кнопка `disabled` + подсказка «Вставьте API-токен Потока». При вводе — «Токен введён ✓».

## Превью импорта (общий шаг для обоих источников)

Над сводкой — **баннер источника:**
- Файл: «Кандидаты из файла `<имя файла>` — проверьте перед заливкой в базу».
- Поток: «💃 Глафира забрала кандидатов из **Потока** по API — проверьте перед заливкой в базу».

### Сводка-полоса (стат-карточки)

| Карточка | Цвет | Смысл |
|---|---|---|
| Всего строк | нейтральный | число записей в источнике |
| Новых кандидатов | зелёный | будут созданы |
| Дублей | жёлтый | уже есть в базе (матч по телефону/email) |
| С ошибками | красный | будут пропущены |

### Тумблер дублей
Сегмент-контрол: **Дубли: Пропустить / Обновить**.
- **Пропустить** — совпавшие с базой не трогаем (создаём только новых).
- **Обновить** — совпавшим обновляем контакты и поля из источника.

Кнопка импорта меняет число динамически: «✓ Импортировать **X** кандидатов».

### Таблица превью
По строке на кандидата (первые ~20–50, с прокруткой): аватар-инициал, имя, телефон, email, город, источник (чип).
- **Дубли** — бейдж «дубль». **Ошибки** — красная пометка + причина («нет имени», «нет контакта»), строка не кликабельна.
- Под таблицей: «и ещё **K** строк будут обработаны при импорте».
- Клик по строке открывает **попап с резюме** (см. ниже).

«Назад» возвращает в ветку источника: к **Колонкам** (файл) или к **Токену** (Поток).

> **Edge case · большой объём.** Превью показывает первые N с прокруткой и счётчиком «и ещё K». Полная обработка — на шаге импорта.

## Импорт и результат (общий шаг)

### Выполнение
Персонаж 💃 + прогресс-бар «Импортировано **N** из **M**…» и процент.

### Результат
Зелёный чек ✓ «Импорт завершён». Подзаголовок зависит от источника («Кандидаты из файла / из Потока добавлены в общую базу») + 3 стат-карточки:
- **Создано X** кандидатов
- **Пропущено / обновлено Y** дублей (зависит от тумблера)
- **Ошибок Z** (пропущены)

Кнопки: **Смотреть в базе** (→ общая база) · **Импортировать ещё** (→ шаг «Источник»). Сноска: ошибочные строки можно поправить — в файле (файл) или в Потоке и подключиться повторно (Поток).

## Попап «Резюме из Потока»

Открывается по клику на строку превью. Центрированный модал поверх экрана с затемнением. Закрытие — крестик или клик по фону.

Содержимое:
- Шапка: аватар + имя кандидата.
- Чипы: «Резюме из Потока», источник (hh.ru / Авито / Telegram), желаемая роль.
- Карточка резюме: должность, «Общий стаж · Город · Последнее место», контакты, блоки **О кандидате**, **Опыт работы** (должности, компании, периоды, описания), **Навыки**, **Образование**, **Дополнительно**.

> **Честность по продукту.** Резюме-ссылки из потоковых систем могут не открываться вне исходной системы. В попапе явная плашка: «Резюме и биография забраны из Потока. Ссылка на исходный файл сохранится в карточке, но сам PDF-файл резюме недоступен вне Потока».

## Распознавание колонок (только ветка «Файл»)

Глафира матчит названия колонок исходной системы на наши поля (без учёта регистра, по синонимам):

| Наше поле | Примеры колонок в выгрузках |
|---|---|
| **Имя** \* | `ФИО Кандидата` · `ФИО` · `Имя` · `Кандидат` · `Full name` |
| **Телефон** | `Моб. телефон` · `Телефон` · `Номер телефона` · `Phone` |
| **Email** | `E-mail` · `Эл. почта` · `Почта` · `Email` |
| Город | `Город` · `Город проживания` · `City` |
| Возраст | `Возраст` · `Age` |
| Зарплата | `Зарплатные ожидания` · `ЗП` · `Ожидания по зарплате` · `Salary` |
| Источник | `Источник` · `Источник отклика` · `Source` |
| Должность | `Желаемая должность` · `Должность` · `Позиция` |
| Комментарий | `Комментарий` · `Комментарий рекрутёра` · `Note` |
| Резюме-ссылка | `Резюме` · `Ссылка на резюме` · `CV link` |
| — не импортировать — | `ID в Потоке` · `Метка` · служебные поля (не распознаются → ручной выбор) |

## Edge cases и валидации

| Ситуация | Поведение |
|---|---|
| Файл не Excel / повреждён | Ошибка чтения, переход дальше заблокирован. |
| Обязательные поля не замаплены (файл) | Кнопка «Далее» (шаг Колонки) `disabled` + подсказка. |
| Пустой токен (Поток) | Кнопка «Подключиться» `disabled` + подсказка. |
| Строка без имени или без контакта | В превью помечается ошибкой с причиной, при импорте пропускается. |
| Дубль с базой | Бейдж «дубль»; поведение задаётся тумблером Пропустить / Обновить. |
| Резюме-файл из другой системы | Сохраняем ссылку, но честно предупреждаем, что файл недоступен вне исходной системы. |
| Большой объём | Превью первых N + «и ещё K»; прогресс-бар на импорте. |

## Нефункциональные требования

- **Форматы файла:** `.xlsx`, `.xls`. (CSV — вне MVP.)
- **Поток:** подключение по API-токену (доступ на чтение кандидатов); забор кандидатов и резюме по API.
- **Объём:** штатно до 5000+ записей; импорт асинхронный, с прогрессом.
- **Дедупликация:** матч по нормализованному телефону и/или email.
- **Локализация:** RU. Числа/даты — `--font-mono` (JetBrains Mono).
- **Дизайн-система:** токены проекта (`styles/tokens.css`) — ark-палитра gray/blue, шрифты Inter + JetBrains Mono, паттерны кнопок, карточек, таблиц, чипов как на других экранах.
- **Источник:** импортированным кандидатам проставляется источник (из файла или «Поток»); служебно — признак «импорт».

## Вне MVP / открытые вопросы

- Поддержка `.csv` и выбора листа/разделителя.
- Сохранение шаблона маппинга для повторных выгрузок из той же системы.
- Реальный парсинг файла резюме (PDF) из ссылки — зависит от доступа к исходной системе.
- OAuth-подключение к Потоку вместо ручного токена; выбор воронок/тегов при заборе.
- **Интеграции Talantix и Хантфлоу** (сейчас заглушки «Скоро») — подключение по API-токену, как Поток.
- Привязка импортируемых кандидатов сразу к вакансии — намеренно **не делаем** (заливаем в базу).
- Правила автоматического слияния дублей (какие поля приоритетнее при «Обновить»).
