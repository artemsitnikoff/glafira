# Экран 07. Пульс — Главный

> Открывается при клике на пункт «Пульс» (💗) в сайдбаре. Раздел работы **с уже нанятыми сотрудниками**.

**Файл компонента:** `components/PulseMain.jsx`
**Связанные:** `PulseDetail.jsx` (карточка сотрудника), `PulseData.jsx` (моки данных)

---

## 1. Назначение раздела

«Пульс» — это пост-найм-модуль. Когда кандидат становится «Нанят» в воронке, он переходит сюда и начинает «жить» как сотрудник в адаптации.

Цели раздела:
- следить, как новые сотрудники проходят онбординг и испытательный срок,
- ловить ранние сигналы ухода (Глафира анализирует ответы на опросы и активность),
- проводить регулярные пульс-опросы (eNPS, удовлетворённость),
- сравнивать команды / руководителей по качеству адаптации.

Это связывает воронку рекрутинга с реальной работой компании.

## 3. KPI-полоса

| Метрика | Описание |
|---|---|
| **На адаптации** | Кол-во сотрудников в первые 90 дней работы |
| **Прошли испытательный** | За период (с дельтой) |
| **Ушли в первые 90 дней** | Абс + % от вышедших (красная дельта плохо) |
| **eNPS** | Средний рекомендательный индекс (от −100 до +100) |

Карточки — кликабельны, ведут в соответствующий отчёт Аналитики (Текучка / Опросы).

## 4. Алерты Глафиры

Если AI-агент засёк сигнал риска — в верхней части экрана появляется горизонтальный список карточек-алертов.

### Виды алертов
- 🔴 «**Иван Петренко** — высокий риск ухода: 2 опроса подряд с низкой оценкой»
- 🟡 «**Мария Корнеева** — пропустила 2 еженедельных опроса»
- 🟡 «**Команда А. Корнеева** — eNPS упал на 15 пунктов за месяц»
- 🔵 «**Сегодня** — 3 пульс-опроса нужно отправить вручную»

### Дизайн карточки
- Цветная полоска слева (красный/жёлтый/синий).
- Иконка + заголовок.
- Краткий контекст.
- Кнопка действия: «Открыть карточку» / «Связаться» / «Запустить опрос».

Алерты можно скрыть (X на карточке) — пропадают до следующего сигнала.

## 5. Таблица сотрудников

Главный список раздела. Похожа по структуре на таблицу кандидатов из Экрана 03, но колонки другие.

### Колонки
| Колонка | Описание |
|---|---|
| ☐ | Чекбокс |
| Аватар + ФИО + должность | Двустрочная |
| Руководитель | ФИО менеджера |
| Дата выхода | Когда начал работать (моно) |
| День адаптации | «День 23 из 90» + полоска прогресса |
| Риск ухода | Цветная плашка `low / mid / high` (см. ниже) |
| Последний опрос | Дата + краткий результат (👍 / 😐 / 👎) |
| Действия | Иконки: 💬 связаться, 📋 запустить опрос |

### 5.1 Полоса адаптации (`AdaptBar` в `PulseDetail.jsx`)
- Тонкая полоска ширины колонки.
- 4 секции: 0–30 / 31–60 / 61–90 / >90.
- Текущая позиция — точка/маркер.
- Цвет полоски совпадает с цветом риска ухода.

### 5.2 Риск ухода
- 🟢 Low: всё хорошо, сигналов нет.
- 🟡 Mid: один тревожный сигнал (пропуск опроса, низкая оценка).
- 🔴 High: 2+ сигналов за неделю.

Алгоритм считает Глафира на основе:
- ответов на пульс-опросы,
- активности в корпоративных каналах (если интегрировано),
- комментариев руководителя,
- соответствия плана адаптации факту.

### 5.3 Сортировки и фильтры
- Сортировка по любой колонке (как в воронке).
- Фильтры: руководитель, отдел, период выхода, риск, статус опроса.
- Поиск по ФИО.

## 6. Поведение

| Действие | Результат |
|---|---|
| Клик по строке | Открывается карточка сотрудника (Экран 08) — справа |
| Клик по «Запустить опрос» | Модал — выбор шаблона опроса + получатели |
| Клик по «Связаться» | Открывает чат с сотрудником в встроенном мессенджере |
| Чекбоксы + bulk | «Запустить опрос для всех», «Отправить сообщение» |

## 7. Сегменты вверху списка (опционально)

Чипы-сегменты для быстрых пресетов:
- `Все` (default)
- `На адаптации` (день ≤ 90)
- `Прошли испытательный` (день > 90)
- `🔴 Высокий риск`
- `Без последнего опроса > 14 дней`
- `Уволенные за период`

## 8. Дизайн-детали

- Цвета риска: `--risk-low` зелёный, `--risk-mid` оранжевый, `--risk-high` красный.
- Аватары — `Avatar` компонент (инициалы цветным кругом).
- Фон строки на hover — `--bg-3`.
- Полоса адаптации — высота 6 px, `border-radius: 3px`, светлый трек + цветная заливка.
