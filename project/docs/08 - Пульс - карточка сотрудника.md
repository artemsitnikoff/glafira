# Экран 08. Пульс — Карточка сотрудника

> Открывается при клике на строку сотрудника в Пульсе (Экран 07). Right-side панель, аналог карточки соискателя из Экрана 04.

**Файл компонента:** `components/PulseDetail.jsx`

---

## 1. Назначение

Полная картина по конкретному сотруднику в адаптации:
- профиль и базовая инфа (откуда пришёл, кто рекрутил, когда вышел),
- прогресс адаптации (план / факт по дням),
- история опросов и их результаты,
- алерты Глафиры по нему,
- чат и комментарии.

Карточка отвечает на вопрос «Что сейчас происходит с этим человеком и нужно ли что-то делать?».

---

## 2. Структура карточки

```
┌──────────────────────────────────────────────────────────┐
│  Тулбар: [💬 Связаться] [📋 Запустить опрос] [✕]          │
├──────────────────────────────────────────────────────────┤
│  Шапка: Аватар + ФИО + должность                         │
│         Руководитель · Отдел · Дата выхода · День N из 90│
│         Полоса адаптации (большая)                       │
│         Риск-плашка                                      │
├──────────────────────────────────────────────────────────┤
│  Алерты Глафиры (если есть)                              │
├──────────────────────────────────────────────────────────┤
│  Табы: Обзор · План адаптации · Опросы · Чат · Действия  │
├──────────────────────────────────────────────────────────┤
│  Контент таба                                            │
└──────────────────────────────────────────────────────────┘
```

---

## 3. Тулбар

| Кнопка | Действие |
|---|---|
| **Связаться** | Открывает чат во внутреннем мессенджере (или прямой TG/WhatsApp) |
| **Запустить опрос** | Модал: шаблон + время отправки |
| **Заметка** | Быстрая внутренняя заметка для руководителя |
| **✕** | Закрыть карточку |

---

## 4. Шапка карточки

### Левая колонка
- Большой аватар (64 px).
- **H1**: ФИО.
- Подпись: должность · отдел.
- Мета-строка: «Руководитель: А. Корнев · Дата выхода: 12.03.26 · День 23 из 90».

### Правая колонка
- **Риск-плашка** — большая, цветная: `🟢 Low risk` / `🟡 Mid risk` / `🔴 High risk`.
- Под ней: «Последний опрос: 7 дней назад · 4/5 ⭐».

### Полоса адаптации (`AdaptBar`)
Большая, на всю ширину шапки:
- 4 секции: 0–30 / 31–60 / 61–90 / >90 (испытательный пройден).
- Текущая точка — крупный маркер.
- Над секциями — мини-метки ключевых событий: «Welcome-pack ✅», «1-on-1 с руководителем ✅», «Промежуточная встреча ⏳», «Финальное ревью ⏳».

---

## 5. Алерты Глафиры

Если на сотрудника есть открытые алерты — они показываются под шапкой:
- 🔴 «2 пульс-опроса подряд с низкой оценкой удовлетворённости» + кнопка «Связаться»
- 🟡 «Не зашёл в корп-канал 5 дней» + кнопка «Спросить»
- 🟡 «Руководитель не провёл 1-on-1 за последние 14 дней» + кнопка «Напомнить руководителю»

Каждый алерт — карточка с цветной полоской и действием.

---

## 6. Табы

5 табов, дефолт — **«Обзор»**.

### 6.1 Таб «Обзор»
- AI-сводка от Глафиры (1 абзац): «Иван проходит адаптацию хорошо. Удовлетворённость 4.2/5. Один тревожный сигнал — пропустил последний опрос».
- 4 мини-плашки:
  - **eNPS** — рекомендательный индекс
  - **Удовлетворённость** — средняя оценка из опросов
  - **Активность** — % дней с активностью
  - **Прогресс плана** — % выполненных пунктов плана адаптации
- Краткий timeline ключевых событий (последние 5).

### 6.2 Таб «План адаптации»
Чек-лист пунктов на 90 дней. Каждый пункт:
- Чекбокс (выполнено/нет)
- Заголовок («Пройти курс по продукту», «Встреча с командой», «Первый код в продакшен»)
- Дедлайн (день N от выхода)
- Ответственный (HR / руководитель / сам сотрудник)

Группировка по фазам: Welcome / Первый месяц / Второй месяц / Третий месяц.

Глафира **сама** генерирует план на основе должности и шаблона компании.

### 6.3 Таб «Опросы»
История пульс-опросов:
- Каждый опрос — карточка: дата, тип (еженедельный / ежемесячный / специальный), результат (общий балл).
- Раскрытие — все вопросы с ответами и комментарием.
- Сравнение с предыдущим опросом и со средним по команде.
- Кнопка «Запустить новый опрос».

### 6.4 Таб «Чат»
Аналог чата из Экрана 04 (карточка соискателя), но для коммуникации с сотрудником. Каналы те же: TG / Max / WhatsApp / SMS / встроенный мессенджер.

### 6.5 Таб «Действия» (timeline)
Хронология всех событий по сотруднику:
- Глафира отправила опрос
- Сотрудник ответил
- Руководитель оставил комментарий
- Глафира заметила алерт
- Сменили статус
- и т. д.

Та же лента, что в карточке соискателя, только события другие.

---

## 7. Связи между Пульсом и Воронкой

В шапке карточки — мини-блок «Откуда пришёл»:
- Источник найма (hh / Авито / реф)
- Рекрутёр, который вёл
- Срок подбора (дни)
- Ссылка «Открыть исходную карточку соискателя» — ведёт в Экран 04 в режиме «архив».

Это нужно, чтобы:
- HR увидел корреляцию «откуда пришёл — как адаптируется»,
- руководитель отдела рекрутинга мог разобраться в случае ухода новичка.

---

## 8. Дизайн-детали

- Карточка — overlay поверх таблицы Пульса. Ширина адаптивная.
- Скрол внутренний, шапка липкая.
- Аватар — 64 px в карточке.
- Полоса адаптации в шапке — высота 12 px, с подписями фаз сверху.
- Риск-плашка — крупная, цветная: фон `--risk-{low|mid|high}-soft`, текст и точка `--risk-{low|mid|high}`.

---

## 9. Состояния

- **Только что вышел (день 1)**: «Иван Петренко начал работать сегодня. Глафира отправит первый welcome-опрос завтра.»
- **Прошёл испытательный**: тулбар добавляет кнопку «Закрыть карточку адаптации» (переводит сотрудника в «обычный» режим, без активного мониторинга).
- **Уволен**: вместо тулбара — баннер «Сотрудник уволен 04.05.26» + причина + ссылка «Открыть exit-интервью».
