# Экран 09. Настройки

> Открывается при клике на пункт «Настройки» (⚙) в нижней части сайдбара.

**Файл компонента:** `components/Settings.jsx`

---

## 1. Назначение

Раздел конфигурации всего, что касается:
- профиля пользователя и команды,
- интеграций с источниками (hh, Авито, Telegram, мессенджеры, календарь),
- поведения Глафиры (тон голоса, скрипты, уровень автономии по умолчанию),
- шаблонов писем и опросов,
- биллинга и подписки.

---

## 2. Структура раздела

3-колоночный layout:

```
┌──────────┬──────────────────┬──────────────────────────────┐
│ Главный  │  Подменю         │  Контент выбранного раздела  │
│ сайдбар  │  настроек        │                              │
│          │  • группа 1      │                              │
│          │    – пункт       │                              │
│          │    – пункт       │                              │
│          │  • группа 2      │                              │
└──────────┴──────────────────┴──────────────────────────────┘
```

Подменю настроек — отдельная панель ширины ~280 px, аналогично Вакансиям.

> **Важно:** в реализации (`Settings.jsx`) подменю организовано как **верхняя горизонтальная панель табов** (`SettingsTopTabs`) для разделов первого уровня — «Профиль», «Общие», «Воронка по умолчанию», «Права доступа», «Теги», «Интеграции», **«AI»** (последним). Текстовые лейблы без «А»-бейджа администратора. Внутри раздела — обычный контент-флоу с карточками.

---

## 3. Подменю настроек

Группы и пункты:

### Группа «Аккаунт»
- **Профиль** — мои данные (ФИО, email, телефон, аватар, пароль).
- **Уведомления** — куда и какие алерты приходят (email, TG, push).

### Группа «Команда»
- **Сотрудники** — список рекрутёров и менеджеров команды; добавление, роли.
- **Роли и права** — настройка матрицы доступов (кто что видит и редактирует).
- **Клиенты / Заказчики** — справочник компаний (для агентств).

### Группа «Интеграции»
- **Источники найма**:
  - hh.ru — ключ API, активные регионы, шаблоны публикаций
  - Авито — токен, лимиты публикаций
  - Telegram — собственный бот Глафиры (имя, аватар, описание)
  - Реф-программа — реферальные ссылки
- **Мессенджеры**:
  - WhatsApp Business — номер, шаблоны
  - Max — авторизация
  - SMS — провайдер, отправитель
- **Календари**: Google / Outlook / Яндекс — для слотов интервью.
- **Корпоративные системы**: 1С / SAP HR — двусторонняя синхронизация (для Пульса).

### Группа «Глафира»
- **Голос и тон** — выбор «личности» AI-агента: формальный / дружеский / деловой. Превью реплик в чате.
- **Скрипты скрининга** — глобальные шаблоны вопросов, по которым Глафира общается с кандидатом. Можно создать свои + назначить дефолт.
- **Правила автодействий** — пороги: «при AI-скоринге < N — отклонять», «после X дней без ответа — завершать». Слайдеры и чекбоксы.
- **Стоп-слова и стоп-факторы** — глобальный блок-лист.
- **Режим работы по умолчанию** — A/B/C для новых вакансий.

### Группа «Шаблоны»
- **Письма** — шаблоны email кандидатам (приглашение на интервью, оффер, отказ).
- **Сообщения в чат** — короткие шаблоны для TG/WhatsApp.
- **Опросы Пульса** — шаблоны eNPS / удовлетворённость / специальные.

### Группа «Биллинг»
- **Тариф и подписка** — текущий план, лимиты, история списаний.
- **Способы оплаты** — карты, счета.
- **Использование** — счётчики (сколько вакансий, кандидатов, токенов AI).

### Группа «Прочее»
- **Общие** — язык, часовой пояс, формат даты.
- **API и вебхуки** — для технической интеграции.
- **Логи и аудит** — кто что менял (для админов).

---

## 4. Подменю-дизайн

- Группа — заголовок мелким серым `font-size: 11px`, `letter-spacing: 0.05em`, `text-transform: uppercase`.
- Пункт — обычная строка с иконкой слева; активный — фон `--bg-active` + цветная полоска слева.
- Поиск сверху подменю — «Поиск по настройкам…» (ищет по всем пунктам, фильтрует список).

---

## 5. Контент-страницы (типовые паттерны)

Все страницы настроек используют одинаковую сетку:

```
┌───────────────────────────────────────────────────────────┐
│  Заголовок раздела          [Действие: Сохранить / Импорт]│
│  Подзаголовок-описание                                    │
├───────────────────────────────────────────────────────────┤
│  Карточка 1                                               │
│  ┌─ Заголовок группы полей                              ─│
│  │  Поле 1: [______________]                             │
│  │  Поле 2: [tog] [______]                               │
│  │  Поле 3: [chip][chip][chip][+]                        │
│  └─                                                      │
├───────────────────────────────────────────────────────────┤
│  Карточка 2                                               │
└───────────────────────────────────────────────────────────┘
```

### Типы контролов
- **Inputs** — текст, email, телефон, пароль (с глазиком).
- **Toggle** — иконка-переключатель для on/off.
- **Slider** — числовые пороги (например, «AI-скоринг для авто-отказа: ниже **45**»).
- **Chips** — мульти-выбор (регионы, города, источники).
- **Select / Combobox** — длинные списки (валюта, часовой пояс).
- **File upload** — загрузка аватара, логотипа компании.
- **Textarea / Rich-text** — шаблоны писем (с подсветкой переменных `{{candidate_name}}`).
- **Code-mirror** — для технических полей (JSON-схемы, регулярки).

## 6. Ключевые экраны (детально)

### 6.1 Профиль
- Аватар (drag-n-drop или иконка кнопки).
- ФИО, email, телефон, должность.
- Часовой пояс.
- Смена пароля (сепаратная карточка с подтверждением).
- Действие: «Сохранить изменения».

### 6.2 Сотрудники команды
- Таблица: аватар + ФИО + email + роль + статус + действия (Редактировать / Деактивировать).
- Кнопка `+ Пригласить` (модал — email + роль + опционально вакансии).
- Фильтр по роли и статусу.

### 6.3 Голос Глафиры
- 3 варианта tone-of-voice как карточки-радио:
  - **Дружеский** — пример сообщения превью
  - **Формальный** — пример сообщения превью
  - **Деловой** — пример сообщения превью
- Дополнительно: «Использовать имя кандидата на ты/вы», «Эмодзи: умеренно/нет», «Подпись от рекрутёра в конце».

### 6.4 Правила автодействий
Список правил в виде «if-then» карточек:
```
Если: AI-скоринг < 45      →  Отклонить автоматически
Если: 7 дней нет ответа    →  Закрыть как "Не вышел на связь"
Если: ЗП ожидания > вилки  →  Перевести в "На рассмотрение"
```
Каждое правило — карточка с двумя селектами (условие / действие). Можно добавлять/выключать. Глобальные пороги — слайдеры в шапке раздела.

### 6.4.1 Воронка по умолчанию (причины отказа)
Глобальный справочник причин, который используется в:
- popover «Отклонить» в карточке соискателя (Экран 04),
- bulk-bar в воронке (Экран 03),
- отчёте «Отказы» в аналитике (Экран 06).

Причины разделены на **две группы**, каждая выглядит как ряд чипсов-пилюль с точкой-индикатором слева и крестиком справа:

- **От кандидата** (синий буллет): «Не вышел на связь», «Не устроила ЗП», «Принял другой оффер», «Не устроил график», «Слишком далеко от дома».
- **Со стороны компании** (красный/серый буллет): «Несоответствие опыта», «Несоответствие навыков», «Не прошёл интервью», «Не прошёл СБ», «Завышенные ожидания по ЗП».

В конце каждой группы — пунктирная чипса **«+ Добавить»** для расширения справочника. Удаление — через крестик на чипсе.

### 6.5 Шаблоны писем
- Список шаблонов (название, тип события, ENABLED toggle).
- При клике — редактор:
  - Тема + тело письма
  - Поддерживаемые переменные: `{{candidate_name}}`, `{{vacancy}}`, `{{recruiter_name}}`, `{{interview_link}}`, `{{salary}}`...
  - Превью с подставленными значениями.
  - Кнопка «Тестовая отправка на мою почту».

### 6.6 Шаблоны опросов Пульса
- Список шаблонов: «Welcome-опрос (день 7)», «Адаптация (день 30)», «eNPS квартальный»…
- Каждый шаблон:
  - Список вопросов с типами (1–5 шкала, 1–10, текст, multi-choice).
  - Условие отправки (день после выхода / каждый N дней).
  - Каналы: email / TG / встроенный.

### 6.7 Биллинг
- Карточка тарифа: текущий план, цена/мес, дата следующего списания.
- Лимиты с прогресс-барами: «Вакансий 12/25», «AI-токенов 480к/1М».
- Кнопка «Изменить тариф» — модал-сравнение планов.
- История платежей (таблица).

### 6.8 Интеграции
Каждый источник — отдельная карточка:
- Лого + название.
- Статус (зелёная точка «Подключено» / серая «Не подключено»).
- Кнопка «Настроить» / «Отключить».
- Список параметров (в раскрытом виде).

### 6.9 AI (искусственный интеллект)

> Таб **«AI»** — последний в `SettingsTopTabs`. Только для администратора (`adminOnly`). Компонент `SettingsAI` в `components/Settings.jsx`. Заголовок раздела — «Искусственный интеллект», подзаголовок — «Семантический поиск по базе и модель, которая анализирует резюме». При изменениях в шапке появляется баннер «Есть несохранённые изменения» + активная кнопка «Сохранить изменения» (общий паттерн `PageHead`).

Два блока-карточки:

**Карточка 1 — «Семантический поиск по базе кандидатов»**
Описание: «Индексация резюме для умного поиска по смыслу, а не только по ключевым словам. Векторы резюме хранятся в системе и используются разделом „Умный подбор“.»

- **Блок статуса индексации** (`.ai-index`, фон-инсет):
  - Две крупные метрики через стрелку: **«Резюме в базе»** (2 847) → **«Проиндексировано · векторов»** (2 731). Числа моноширинным, формат `ru-RU` с разделителем-пробелом.
  - Пилюля статуса справа: «N в очереди» (серая) либо «Всё проиндексировано» (зелёная, с галочкой), во время процесса — «Индексируется…».
  - **Прогресс-бар** (`.ai-progress`) с процентом готовности; во время переиндексации — анимированный шиммер.
  - Подпись: процент + «Последняя индексация: <дата> · хранилище векторов 412 МБ».
  - **Кнопка «Переиндексировать»** (`.btn-primary`, иконка refresh). По клику запускает анимацию полной переиндексации: счётчик обнуляется и добирается до 100 %, кнопка переходит в состояние «Переиндексация…» и блокируется. Подпись рядом: «Новые резюме индексируются автоматически. Полная переиндексация нужна после смены настроек.»
- **Модель эмбеддингов** (`FormRow`, зафиксирована):
  - Значение `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` моноширинным, иконка-бейдж базы данных, пилюля **«Зафиксирована»** с замком (нередактируемо).
  - Подсказка: «Преобразует текст резюме в векторы. Многоязычная модель, оптимизированная под русский язык. Зафиксирована.»

**Карточка 2 — «Модель LLM»**
Описание: «Настройка модели искусственного интеллекта для анализа резюме: квалификация откликов, оценка соответствия вакансии и саммари кандидата.»

- **Основная модель** (`FormRow` → `Select`, `.ai-model-select`): иконка-бейдж CPU + выпадающий список. Варианты:
  - Qwen3.7-Max
  - Kimi K2.6
  - DeepSeek V4 Flash
  - **Claude Sonnet 4.6** — выбрана по умолчанию
  - Подсказка: «Модель, которая оценивает резюме и квалифицирует отклики. Смена влияет на все новые анализы.» Смена значения помечает раздел как изменённый (баннер + кнопка «Сохранить»).

**Связи:**
- **Умный подбор (11)** — использует векторы резюме, построенные семантической индексацией.
- Выбранная LLM-модель влияет на AI-скоринг и саммари кандидатов во всех экранах, где Глафира анализирует резюме.

**CSS:** классы `.ai-index`, `.ai-index-stats`, `.ai-stat`, `.ai-progress`/`.ai-progress-fill(.busy)`, `.ai-model-locked`, `.ai-model-select`, `.ai-soon-pill`, `.ai-prov-ic` — в `styles/settings.css`. Иконки `database`, `cpu`, `lock` добавлены в `components/primitives.jsx`.

## 7. Дизайн-детали

- Контент — макс. ширина ~880 px (читабельно), центрирование.
- Между секциями карточек — 24 px зазор.
- Кнопка «Сохранить» — sticky в подвале (если страница длинная).
- Изменения подсвечиваются жёлтой точкой у поля и баннером «Не забудьте сохранить».
- Отмена — кнопка `Отменить` сбрасывает к последнему сохранённому состоянию.

---

## 8. Состояния

- **Загрузка** — скелетоны на полях.
- **Сохранение** — спиннер на кнопке.
- **Ошибка** — красная полоска под полем + текст ошибки.
- **Успех** — toast «Сохранено».
- **Нет прав** — баннер «У вас нет прав на редактирование. Обратитесь к админу команды.»

---

## 9. Связи

- **Сайдбар (10)** — точка входа.
- **Главная (01)** — пустые состояния ведут в первую установку (онбординг).
- **Все экраны** — настройки влияют глобально (тон Глафиры, шаблоны, режимы).
