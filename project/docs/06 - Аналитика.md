# Экран 06. Аналитика

> Открывается при клике на пункт «Аналитика» (📊) в сайдбаре.

**Файлы компонентов:** `components/Analytics.jsx` (shell + отчёты 1–3), `components/AnalyticsReports.jsx` (отчёты 4–7), `components/AnalyticsCharts.jsx` (визуализации).

---

## 1. Назначение

Раздел отчётности для рекрутёра, нанимающего менеджера и руководителя HR-отдела. 7 готовых отчётов закрывают типовые вопросы:

- Что вообще происходит? (Обзор)
- Где затыки во времени? (Скорость)
- Где затыки в воронке? (Воронка)
- Откуда лучшие кандидаты? (Источники)
- Почему уходят? (Отказы)
- Кто из новичков уходит? (Текучка)
- Кто из рекрутеров эффективнее? (Рекрутеры)

Аналитика **не редактируется** пользователем — это готовый набор отчётов с фильтрами по периоду / вакансиям / рекрутёрам.

## 3. Глобальные фильтры (шапка)

Над всеми отчётами:
- **Период** — сегментный переключатель: `Неделя · Месяц · Квартал · Год · Произвольный`. Произвольный — открывает date-picker.
- **Вакансия / Клиент / Рекрутёр** — мульти-селекты (опционально, не во всех отчётах).
- **Сравнение с прошлым периодом** — toggle on/off (по дефолту on).
- **Экспорт** — кнопка `Скачать в XLSX / PNG`.

## 4. Отчёт 1. Обзор

Stripped-down дашборд по разделу — общая сводка.

### KPI-полоса (5 метрик)
| Метрика | Описание |
|---|---|
| Активных вакансий | Сейчас в работе |
| Откликов за период | Сумма из всех источников |
| Закрыто вакансий | Кол-во найма за период |
| Средний срок закрытия | Дни от публикации до выхода |
| Стоимость найма | ₽ / выход (если включён биллинг) |

Каждая карточка — крупное число + дельта vs прошлый период.

### Графики
1. **Динамика откликов** (линия) — день/неделя.
2. **Карта активности воронки** — полоса с распределением по этапам (как в Экране 03, но агрегировано по всем вакансиям).
3. **Top-5 вакансий по откликам** — горизонтальный бар-чарт.

---

## 5. Отчёт 2. Скорость

Где «зависают» кандидаты.

### Метрики
- Среднее время на каждом этапе воронки (бар-чарт по этапам).
- p50 / p90 для каждого этапа (показывает «хвосты»).
- Средний `time-to-hire` в разрезе вакансий (таблица).
- Сравнение со среднерыночным (если включена бенч-функция).

### Визуализация
- **Box-plot** по этапам — медиана + квартили + outliers.
- **Heatmap «вакансия × этап»** — показывает, где конкретно в каждой вакансии затык. Цветом — время в днях.

---

## 6. Отчёт 3. Воронка

Конверсия из этапа в этап.

### Графика
- **Воронка** (классический funnel) — высота полосы пропорциональна количеству, проценты — конверсия из предыдущего.
- Под каждым этапом: число кандидатов + конверсия.
- Терминальные исходы внизу: «Нанят N (M%)» / «Отказ N (M%)».

### Таблица
Конверсии в разрезе:
- источника (hh / Авито / Telegram)
- вакансии
- рекрутёра

Чтобы увидеть — кто и где конвертит лучше.

## 7. Отчёт 4. Источники (Sources)

**Файл:** `AnalyticsReports.jsx` → `ReportSources`.

Какой источник приводит лучших кандидатов.

### Таблица «Эффективность источников»
| Колонка | Описание |
|---|---|
| Источник | hh.ru / Авито / Глафира · Telegram / Реф-программа / Прямой поиск |
| Откликов | всего за период |
| Прошли скрининг | абс + % |
| Дошли до интервью | абс + % |
| Нанято | абс + % |
| Стоимость | ₽ за публикацию / лида |
| ROI | условный показатель «качество × конверсия» |

Сортировка по любой колонке. Цветовые маркеры на ROI.

### Графики
- **Stacked bar** «отклики → найм» по источникам.
- **Scatter** «качество (avg AI-скоринг) × объём».

## 8. Отчёт 5. Отказы (Rejections)

**Файл:** `AnalyticsReports.jsx` → `ReportRejections`.

Почему отказываем мы и почему отказывают нам.

### Сегментация
2 колонки:
- **Отказы с нашей стороны** — Глафира + рекрутёры
- **Отказы кандидатов** — отказались сами

### Категории причин (chips/pie)
**Наши:**
- Не соответствует ЗП ожиданиям
- Недостаточно опыта
- Стоп-фактор (география / медкомиссия / ...)
- Не прошёл интервью
- Дубль из базы

**Кандидата:**
- Принял другой оффер
- Передумал работать
- Не вышел на связь
- Не устроили условия

### Визуализация
- Pie-charts по обеим сторонам.
- Динамика во времени — линия.
- Топ-вакансии по отказам — таблица.

## 9. Отчёт 6. Текучка / Адаптация (Turnover)

**Файл:** `AnalyticsReports.jsx` → `ReportTurnover`.

Связан с разделом «Пульс» — отслеживает уход новых сотрудников.

### Метрики
- Доля «прошли испытательный» / «ушли в первые 30/60/90 дней».
- Средний срок жизни нового сотрудника по вакансии / руководителю / источнику.
- Cohort-таблица: «выходы по месяцам × доля оставшихся через N дней».

### Графики
- **Cohort heatmap** — классическая retention-таблица с цветами.
- **Survival curve** — линия «доля оставшихся» во времени.
- **Топ-руководителей по текучке** — таблица.

Этот отчёт читает данные из Пульса (см. Экран 07): кто, когда вышел, кто ушёл.

## 10. Отчёт 7. Рекрутёры (Recruiters)

**Файл:** `AnalyticsReports.jsx` → `ReportRecruiters`.

Эффективность каждого рекрутёра в команде.

### Таблица «Лидерборд»
| Колонка | Описание |
|---|---|
| Рекрутёр | Аватар + ФИО |
| Активных вакансий | сейчас |
| Откликов обработано | за период |
| Скринингов проведено | (Глафира + рекрутёр) |
| Интервью | назначено / проведено |
| Найма | количество |
| Среднее время | до найма |
| Доля автономии Глафиры | % действий, сделанных AI без человека |

Сортировка по любой колонке. Топ-3 — золотыми/серебряными/бронзовыми бейджами.

### Графики
- **Bar-chart** «найма по рекрутёрам».
- **Radar-chart** «многомерное сравнение топ-3 рекрутёров».

## 12. Дизайн-детали

- Цвета графиков: единая палитра воронки (см. STAGES в Candidates.jsx) + нейтральные синий/зелёный/красный для бенчмарков.
- Числа во всех таблицах — `--font-mono`.
- Дельты: `+12%` зелёным, `−5%` красным.
- KPI-карточки — те же, что на главной (унифицированный компонент).
- Фон отчётов — `--bg-1` (странице), карточки — `--bg-2`.
- Графики — SVG, без 3D, без теней. Подписи осей серым 11px моно.

## 13. Состояния

- **Нет данных за период** — illustration + «За выбранный период данных недостаточно. Попробуйте увеличить период».
- **Отдельный отчёт грузится** — скелетоны на KPI и графиках.
- **Экспорт** — toast «Отчёт сохранён».

## 14. Связи

- **Главная (01)** — KPI-карточки оттуда ведут сюда с пресетом периода.
- **Воронка (03)** — клик по конкретной вакансии в отчётах.
- **Пульс (07)** — отчёт «Текучка» использует данные из Пульса.
- **Настройки (09)** — конфиг периодов, биллинг (для стоимости найма).
