# Экран 01. Главная (Dashboard)

> Открывается при клике на пункт «Главная» в сайдбаре. Стартовый экран после входа в систему.

**Файл компонента:** `components/Home.jsx`
**Роль в навигации:** дефолтный экран приложения, точка возврата для рекрутёра.

---

## 1. Назначение экрана

Главная — это операционный дашборд старшего рекрутёра. Здесь он:

- одним взглядом понимает «как идут дела» через 6 ключевых KPI,
- видит, **что требует его внимания** (вакансии, где залип процесс или поджимает срок),
- следит за **лентой событий** в реальном времени (что Глафира и команда сделали за последние часы),
- понимает, **откуда приходят кандидаты** (топ-источники).

Экран — командный пункт. Здесь не работают глубоко: посмотрел → провалился в нужную вакансию или отчёт.

---

## 2. Структура страницы

```
┌──────────────────────────────────────────────────────────────┐
│  Заголовок «Главная» (или «Привет, Анна» — тык)              │
│  Подпись: Обновлено 5 мин назад · 14:32                      │
│                  [Неделя · Месяц · Квартал · Год · Всё время]│
├──────────────────────────────────────────────────────────────┤
│  KPI-сетка (6 базовых, +2 расширенных по тык)                │
├──────────────────────────────────────────────────────────────┤
│  Требуют внимания (≈50%)   │  Лента событий · live (≈50%)    │
├──────────────────────────────────────────────────────────────┤
│  Последние сообщения · все каналы в одном чате               │
├──────────────────────────────────────────────────────────────┤
│  Топ-источники кандидатов (горизонтальные бары) — опционально│
└──────────────────────────────────────────────────────────────┘
```

---

## 3. Шапка экрана

### 3.1 Заголовок
- **«Главная»** — строгий вариант (по умолчанию).
- **«Привет, Анна»** — приветствие по имени из профиля; включается тыком `greeting`.
- Подпись слева внизу: «Обновлено 5 мин назад · 14:32» (моно-цифры, серый).

### 3.2 Переключатель периода
Сегментный контрол справа с 5 кнопками: `Неделя` · `Месяц` · `Квартал` · `Год` · `Всё время`.
- По умолчанию: **Месяц**.
- Управляет данными KPI и подписями «к прошлому …».
- Не влияет на блоки «Требуют внимания», «Лента событий», «Адаптация», «Топ-источники» (там свой контекст).

---

## 4. KPI-сетка

Базовый набор — **6 карточек** в адаптивной grid-сетке (3×2 на широком, 2×3 на среднем):

| # | Метрика | Значение | Подпись/дельта |
|---|---|---|---|
| 1 | Открытые вакансии | число (например, 23) | дельта (`+3`) и «всего активных» |
| 2 | Закрытые вакансии | число (18) | «N успехом · M без найма» |
| 3 | Среднее время найма | дни (31) | дельта со знаком хорошо/плохо + «к прошлому …» |
| 4 | Текучесть (90 дней) | % (6.2) | дельта (▼ 0.8 п.п.) |
| 5 | Активных кандидатов | число (412) | «из них N новых» |
| 6 | Конверсия отклик → найм | % (12.4) | дельта в п.п. |

### Расширенный набор (тык `kpiExtended`)
К базовым добавляются:
- **Стоимость найма (CPH)** — `41 200 ₽`, дельта в %.
- **Скорость отклика рекрутера** — `4.1 часа`, дельта в %.

### Дизайн карточки KPI
- Лейбл сверху, рядом мелкая `i`-иконка-тултип с пояснением.
- Большое число (`--font-mono`, ~40 px), при наличии единица измерения (`%`, `дней`, `₽`, `часа`) — рядом мелкой.
- Низ карточки в две колонки: дельта-чип слева (зелёный/красный/нейтральный с символами ▲▼) + подпись справа (`к прошлому месяцу` / «всего активных» и т. п.).
- Карточка некликабельна сама по себе (пока). Это смотровое стекло.

### Логика дельт
- Для метрик «больше — лучше» (конверсия, наняли): рост = зелёный (`up`), падение = красный (`down`).
- Для метрик «меньше — лучше» (срок найма, текучесть, CPH): падение = зелёный (`down-good`), рост = красный (`up-bad`).
- Нулевая дельта — нейтральный серый (`flat`).

---

## 5. «Требуют внимания» (левая половина)

Список вакансий, где процесс залип или поджимает срок. Это **не задачи человеку лично** — это сигналы по вакансиям.

### Строка
```
[флажок-иконка]  Frontend-разработчик (Senior)
                 Топ-3 кандидата без движения 9 дней            [→]
```

### Виды флажков
- **`urgent`** (красный) — что-то горит: «12 новых откликов не обработано >24ч».
- **`warn`** (жёлтый) — затык: «Топ-3 кандидата без движения 9 дней», «Заказчик не отвечает 3 дня».
- **`deadline`** (оранжевый) — близкий срок: «Дедлайн закрытия через 4 дня».

### Шапка блока
- Слева заголовок «Требуют внимания» + пилюля-счётчик (`4`).
- Справа кнопка-ghost «Все вакансии →» (ведёт в раздел Вакансии).

Клик по строке → воронка соответствующей вакансии (Экран 03).

---

## 6. «Лента событий · live» (правая половина)

Хронологический фид всех значимых событий в системе за последние часы.

### Виды событий
| `type` | Цвет иконки | Примеры |
|---|---|---|
| `qual`  | синий-бренд | «Глафира квалифицировала кандидата X на вакансию Y», «Глафира отклонила Z — не подходит по стажу» |
| `new`   | зелёный | «Новый отклик на вакансию X — Игорь П.», «3 новых отклика на …» |
| `score` | золотой | «Заказчик «Логос» оставил оценку 5/5 кандидату …» |
| `offer` | фиолетовый | «Кандидат принял оффер», «вышел на испытательный» |
| `move`  | серый | «Виктор Л. переведён на этап «Интервью с заказчиком»» |

### Дизайн строки
- Круглая иконка слева (12 px), цвет соответствует типу.
- Текст: имя `«Глафира»` подсвечено бренд-акцентом (класс `.anatoly`); кандидаты, вакансии, заказчики — линки (класс `.ent`).
- Время справа мелким серым (`5 мин назад`, `1 ч назад`).

### Шапка блока
- Заголовок «Лента событий».
- Справа маленький бейдж `live` (с пульсирующей точкой) — индикатор реального времени.

### Поведение
- Контейнер с собственным скроллом (`maxHeight: 380px`), фид сам прокручивается.
- Группировки по дням пока нет — лента «свежий слой» (последние ~2–4 часа).

## 7. «Последние сообщения» (во всю ширину)

Единая лента входящих сообщений рекрутёра по **всем каналам сразу** (Telegram, WhatsApp, hh.ru, Авито, СМС). Глафира собирает переписку из всех мессенджеров и площадок в один поток — рекрутёру не нужно держать пять вкладок открытыми. Снимок последних диалогов; полная работа с перепиской — в табе «Чат» карточки кандидата (Экран 04).

### 7.1 Шапка
- Заголовок «Последние сообщения» + светло-серый подзаголовок `· все каналы в одном чате`.
- Справа кнопка-ghost «Все чаты →».

### 7.2 Сводка-строка
Под шапкой — короткая сводка: `N непрочитанных` · `M активных диалогов`. Цифры моно, акцентом.

### 7.3 Список сообщений
Каждая строка — это диалог с **реальным кандидатом** (все входящие уже сопоставлены с карточкой; «неопознанных контактов» в ленте нет):

- **Аватар** с инициалами кандидата; в правом нижнем углу — цветной бейдж канала (`TG` / `WA` / `HH` / `AV` / `СМ`) в фирменном цвете.
- **Имя кандидата** + пилюля-вакансия, к которой относится диалог.
- **Название канала справа** — в фирменном цвете канала, с цветной точкой (как в самом чате): Telegram — синий, WhatsApp — зелёный, hh.ru — красный, Авито — бирюзовый, СМС — серый.
- **Превью текста** последнего сообщения (1 строка, обрезается).
- **Действие** — ссылка `Перейти к кандидату →` (ведёт в карточку соискателя). Ручной привязки контактов нет — она убрана как избыточная.
- Справа: точка непрочитанного + время (`2 мин`, `1 ч`).

### 7.4 Поведение
- Клик «Перейти к кандидату» / по строке → карточка соискателя (Экран 04), таб «Чат».
- Клик «Все чаты →» → полный список диалогов.
- Блок всегда виден (не за тыком) — входящая коммуникация так же важна, как воронка.

---

## 8. «Топ-источники кандидатов» (опционально)

Включается тыком `showSources` (по умолчанию **включён**).

Горизонтальные бары: hh.ru, Авито Работа, Telegram-бот Глафиры, Рефералы, Прямые отклики. Каждый источник — свой цвет (бренды-источники: красный hh, зелёный Авито, синий Telegram-бот). Длина бара пропорциональна абсолютному значению, число справа моно.

В шапке блока — текущий период мелким серым справа.

---

## 9. Тыки (Tweaks-панель)

| Ключ | Эффект |
|---|---|
| `greeting`     | Заголовок: «Привет, Анна» вместо «Главная» |
| `kpiExtended`  | Добавляет CPH и Скорость отклика к KPI |
| `showSources`  | Показывает блок «Топ-источники кандидатов» |

---

## 10. Поведение и интеракции

| Действие | Результат |
|---|---|
| Смена периода | Перезагружаются числа и дельты в KPI |
| Клик «Все вакансии» | Переход в раздел Вакансии |
| Клик по строке «Требуют внимания» | Воронка соответствующей вакансии (Экран 03) |
| Клик «Все чаты» в блоке «Последние сообщения» | Полный список диалогов |
| Клик «Перейти к кандидату» в ленте сообщений | Карточка соискателя, таб «Чат» (Экран 04) |
| Клик по сущности в ленте (имя кандидата/вакансии) | Карточка соискателя / воронка |
| Hover на `i` в KPI | Тултип с расшифровкой метрики |

---

## 11. Дизайн-токены

- Фон страницы: `--bg-1`; карточки: `--bg-2`, граница `--border-1`, скругление 12 px.
- Цифры — `--font-mono` (Inter Tight Mono или эквивалент).
- Дельта-чипы — мелкие пилюли с фоном-тинтом (`up` → зелёный тинт, `down-good` → зелёный тинт, `up-bad` → красный тинт).
- Брендовый цвет (для подсветки имени «Глафира» в ленте) — `--brand-accent`.
- Иконки — outline-стиль из локального набора `Icon`.

---

## 12. Состояния экрана

- **Загрузка** — скелетоны на KPI и в обоих блоках-листах.
- **Пустое (новый аккаунт)** — KPI-нули, блоки «Требуют внимания» и «Лента событий» — с короткими empty-state стейтами; CTA «Создайте первую вакансию».
- **Период «Всё время»** — все дельты подменяются на `—` (сравнивать не с чем).

---

## 13. Что НЕ делается на этом экране

- Не создаются вакансии и кандидаты (только переход).
- Нет фильтров KPI по сегментам/командам — для этого есть Аналитика.
- Нет глубокой аналитики и графиков — только моментальный срез.
