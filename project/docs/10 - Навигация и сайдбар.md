# Экран 10. Навигация и сайдбар

> Постоянный левый сайдбар, видимый на всех экранах. Главная точка перехода между разделами.

**Файл компонента:** `components/Sidebar.jsx`

---

## 1. Назначение

Сайдбар — это **скелет навигации** приложения. Он:
- даёт быстрый доступ ко всем разделам,
- разворачивает подменю (Вакансии) inline,
- показывает идентичность пользователя (аватар + имя внизу),
- содержит вспомогательные действия (поиск, уведомления, настройки).

Сайдбар **не сворачивается** до иконок — он всегда с подписями. Это сознательный выбор: продукт ориентирован на рекрутёров, для которых не важна экономия 200 px на десктопе, но важна узнаваемость пунктов.

## 3. Шапка сайдбара

- **Бренд-марка** — круглая плашка с эмодзи 👩🏻 (`brand-mark` + `brand-emoji`).
- **Wordmark «Глафира»** — рядом с маркой.
- **Эмодзи 💃** (`brand-dancer`) — небольшая брендовая фигурка справа, удерживает игривость.
- Клик по шапке ведёт на Главную (опционально).

> Глобального поиска (Cmd-K-палитры) и бейджа окружения в текущей реализации **нет** — оставлены как идея на будущее.

## 4. Основная навигация

### Пункты (в порядке)
| Иконка | Лейбл | Куда ведёт | Особенности |
|---|---|---|---|
| `home` | Главная | Экран 01 | Дефолтный |
| `briefcase` | Вакансии | Экран 02 | **Раскрывается inline** (vacanciesOpen) |
| `users` | Кандидаты | Экран 05 | Общая база (все когда-либо отобранные) |
| `chart` | Аналитика | Экран 06 | **Раскрывается inline** (analyticsOpen) — список отчётов |
| `heart` | Пульс-Онбординг | Экран 07 | Бейдж `pip` с числом активных алертов |
| `settings` | Настройки | Экран 09 | |

### Дизайн пункта
- Высота 40 px.
- Иконка слева (20 px), лейбл рядом.
- Активный пункт — фон `--bg-active`, цветная полоска слева 4 px (бренд-акцент).
- Hover — фон `--bg-3`.
- Бейдж справа (если есть) — мелкий, цветной (красный для алертов, синий для счётчиков).

## 5. Раскрытие «Вакансии»

Когда пользователь кликает по пункту **«Вакансии»** (или по шеврону рядом):
1. Под пунктом разворачивается inline-блок `sub-block`.
2. Шеврон поворачивается ▸ → ▾ (анимация на классе `nav-chev.open`).
3. Главный пункт остаётся подсвечен (active).

### Содержимое
- Кнопка «+ Новая вакансия» (открывает форму создания в контентной области).
- Поле поиска (фильтрует список ниже в реальном времени).
- Список вакансий: точка непрочитанных + название + счётчик + пилюля «новых».
- Разделитель.
- Строка «Архив» с числом архивных.

### Поведение
- Раскрытие/сворачивание само по себе **не открывает** воронку — нужно кликнуть по конкретной вакансии, по «Архив» или по «+ Новая вакансия».
- Если активный раздел не «Вакансии», а пользователь кликает по пункту «Вакансии» — раздел становится активным, блок открывается, контентная область заменяется на пустой стейт / первую вакансию / архив (по тыку `defaultSelected`).
- Если уже в «Вакансиях» — повторный клик переключает только видимость блока (`vacanciesOpen`), контент справа не меняется.

## 6. Раскрытие «Аналитика»

Аналогично «Вакансиям»: пункт `Аналитика` тоже раскрывается inline и показывает плоский список отчётов из `window.AN_REPORTS` (Обзор, Скорость найма, Воронка, Источники, Причины отказов, Текучка после найма, Рекрутёры).

- Каждая строка — иконка отчёта + лейбл.
- Активный отчёт подсвечен (`selected`).
- Клик меняет `analyticsReportId` и открывает соответствующий отчёт в контентной области (Экран 06).

## 7. Низ сайдбара (`user-card-wide`)

В одну плашку:
- Аватар пользователя (Анна Седова, инициалы «АС», ~28 px).
- Имя + роль («Старший рекрутер») двумя строками.
- Иконка-кнопка колокольчика справа с маленьким `pip` для индикации непрочитанных.

> Поповера профиля и развёрнутой ленты уведомлений в текущей реализации нет — это задача для следующих итераций.

## 8. Состояния и поведение

| Состояние | Описание |
|---|---|
| Активный экран | Подсвечен соответствующий пункт сайдбара |
| «Вакансии» развёрнуты | Шеврон ▾, `sub-block` виден |
| «Аналитика» развёрнуты | Шеврон ▾, список отчётов виден |
| Внутри вакансии (Экран 03) | Активны и пункт «Вакансии», и конкретная вакансия в списке |
| Pulse-алерты есть | На пункте «Пульс-Онбординг» — бейдж с числом |
| Свёрнутый сайдбар (узкий режим) | **Не реализован** — продукт десктоп-first |

## 9. Дизайн-токены

- Ширина сайдбара (`sidebar-wide`): **240 px** (фиксированная).
- Фон: `--bg-2`, отделён от контента 1 px `--border-1`.
- Иконки: набор `Icon` (`home`, `briefcase`, `users`, `chart`, `heart`, `settings`, `plus`, `search`, `archive`, `chevD`, `bell`).
- Брендовый акцент (полоска активного пункта): `--brand-accent` (мягкий розово-персиковый).
- Шрифт лейблов: `--font-sans`, 14 px, средний вес.
- Бейджи (`nav-row-pip`): мелкие плашки с фоном-тинтом для счётчиков (например, активные алерты Пульса).

---

## 10. Адаптивность

- **≥ 1280 px**: сайдбар всегда виден, ширина 240 px (текущая реализация).
- **< 1280 px**: дополнительно ничего не делается — приложение десктоп-first. Адаптив для планшета/мобайла остаётся как задача.

---

## 11. Связи

Сайдбар — единственный «всегда видимый» элемент UI. Все остальные экраны (01–09) — это контент справа от него.

Из сайдбара:
- открывается любой раздел приложения,
- открывается командная палитра (быстрый доступ к чему угодно),
- доступны общие действия (уведомления, настройки, профиль).
