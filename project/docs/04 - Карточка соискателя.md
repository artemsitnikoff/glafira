# Экран 04. Карточка соискателя

> Не самостоятельный экран — это **right-side панель**, которая выезжает в воронке (Экран 03) при клике на кандидата. Также может открываться из общей базы (Экран 05).

**Файл компонента:** `components/Candidates.jsx` (`CandidateDetail`)

---

## 1. Назначение

Полный профиль кандидата + все каналы коммуникации с ним + история работы рекрутёра и Глафиры. Это место, где принимаются решения «нанимаем / отказываем / двигаем дальше».

Принципиально открывается **рядом со списком**, а не на отдельном экране — рекрутер должен видеть оба окна и быстро переключаться между кандидатами без потери контекста.

## 3. Лоадер (`cd-loading`)

При открытии карточки 850 мс показывается оверлей:
- 💃 эмодзи (танцующая девушка — фирменный персонаж Глафиры)
- Текст «Глафира собирает профиль…» с анимированными точками `cd-load-dots`

Это дизайн-приём, не задержка ради задержки: визуально подтверждает, что AI-агент **сам** работает — собирает данные с hh, парсит резюме, считает скоринг.

## 4. Тулбар (`cd-toolbar`)

3 действия слева + кнопка закрытия справа:

| Кнопка | Тип | Что делает |
|---|---|---|
| **Перевести ▾** | `btn-success` | Открывает popover «На какой этап?» со всеми не-терминальными этапами. Текущий этап подписан «сейчас», следующий — «далее». Из общей базы (`fromPool`) — попап превращается в выбор вакансии. |
| **Отклонить ▾** | secondary | Открывает popover с двумя группами причин: **От кандидата** (5 пунктов с синим буллетом) и **Со стороны компании** (5 пунктов с серым/красным буллетом). Тот же справочник, что в bulk-bar и в Настройках. |
| **Комментарий** | secondary | Скроллит к табу «Комментарии» и фокусирует поле ввода. |
| **ПдН** | secondary (с иконкой) | Видна, если согласие ещё не подписано. Отправляет кандидату запрос на согласие 152-ФЗ через выбранный канал. |
| **«ПдН ✓»** (плашка) | зелёная плашка | Показывается вместо кнопки «ПдН», если согласие уже подписано. Не кликабельна. |
| **✕** | icon-btn | Закрывает карточку. |

### Индикатор ПдН в списке
В таблице кандидатов рядом с ФИО у тех, кто подписал согласие, отображается компактный зелёный бейдж **«✓ ПдН»** (`.pdn-badge`). В шапке карточки — такой же бейдж, размер чуть крупнее.

## 7. Таб «Резюме» (`ResumeTab`)

### 7.1 AI-вердикт-карточка (`AIVerdictCard`)
В самом верху, всегда видна. Это «короткое мнение Глафиры»:

```
┌─────────────────────────────────────────────────────┐
│ 👩🏻  Оценка от Глафиры                          [92] │
│      Хорошо подходит. Релевантный опыт, ключевые    │
│      навыки совпадают с требованиями вакансии.      │
│                                                     │
│  Посмотреть подробную оценку →                      │
└─────────────────────────────────────────────────────┘
```

- Эмодзи 👩🏻 — фирменный персонаж.
- Краткий вердикт зависит от скоринга (≥80, 50–79, <50).
- Линк ведёт в таб «Оценка AI».

### 7.2 Опыт работы
Список мест работы. Каждая запись:
- Должность (полужирный)
- Компания / подразделение
- Период (справа, серый)
- Описание обязанностей и достижений

### 7.3 Навыки
Чипы навыков (`skill-chip`): «B2B продажи», «CRM», «Переговоры», «Английский B2» и т. д. Совпадающие с требованиями вакансии — могут быть подсвечены (если включено).

### 7.4 Образование
Аналогично опыту: вуз / специальность + годы.

### 7.5 Дополнительно (`extra-grid`)
Сетка 2x2 пар:
- Языки: Русский · English B2
- Переезд: не готов
- Командировки: раз в месяц
- Удалёнка: предпочитает

Под сеткой — отдельной строкой во всю ширину **«Предпочтительный способ связи»**: три tap-чипа `Telegram` · `Почта` · `Телефон` с иконками (paper-plane / конверт / телефонная трубка). Один активный (выделен синим тинтом + синей рамкой), остальные — нейтральные. Рекрутёр кликом переключает; значение запоминается в карточке кандидата и подсказывает Глафире и коллегам, через какой канал инициировать контакт. По умолчанию активен `Telegram`.

## 8. Таб «Оценка AI» (`AITab`)

Полная развёрнутая оценка от Глафиры. Структура:

1. **Большой балл** + лейбл «Подходит / Подходит частично / Не подходит».
2. **Вердикт** — 1–2 абзаца текстом.
3. **Сильные стороны** — список с 🟢 пунктами.
4. **Слабые стороны / риски** — список с 🟡 / 🔴 пунктами.
5. **Соответствие требованиям вакансии** — таблица:
   | Требование | Статус | Комментарий |
   |---|---|---|
   | Опыт в B2B ≥ 3 лет | ✅ | 6 лет в Сбере и Альфе |
   | Английский B2+ | ⚠️ | заявлен B2, не подтверждён |
   | Готовность к переезду | ❌ | живёт в Москве, не готов |
6. **Прогноз срока выхода**: «Готов выйти через 4 недели (ожидает оффер от другой компании)».

## 8.5. Таб «Верификация» (`VerifyTab`)

Проверка кандидата по государственным реестрам и публичным источникам. **Доступен только после подписания ПдН** — иначе показывается замок и сообщение «Верификация недоступна» с кнопкой «Запросить ПдН».

Сверху — мета-строка: дата проверки, номер согласия (`PD-XXX/26`), кнопка «Перепроверить».

Все блоки оформлены одинаково: иконка-литера слева, заголовок, цветные чипсы источников (синий — API, фиолетовый — гос-реестр, оранжевый — публичный, красный — AI), статус справа (зелёный «Чисто» / синий «Инфо» / оранжевый «Внимание» / красный «Риск»).

### Блоки проверки

1. **ИНН — идентификация.** Источники: ФНС `service.nalog.ru/inn.do`, DaData. Показывает ФИО / ДР / найденный ИНН / тип совпадения. Fallback: если автопоиск не нашёл — рекрутёр через чат просит кандидата ввести ИНН вручную (Глафира подставит шаблон сообщения).
2. **Исполнительные производства.** Источник: Datanewton (ФССП). Открытые производства, статусы.
3. **Банкротство и связи с юрлицами.** Источники: ЕФРСБ + ЕГРЮЛ/ЕГРИП через DaData. Список юрлиц, в которых кандидат фигурирует как учредитель/директор; признаки банкротства.
4. **Реестры и санкции.** Чек-лист по: реестр дисквалифицированных (ФНС), массовых директоров, статус самозанятого, санкционные списки (РФ / ЕС / OFAC), `kad.arbitr.ru`. Каждый пункт — зелёная галочка либо предупреждение.
5. **Публичная экспертиза.** GitHub, Habr Career, Stack Exchange, TGStat. Глафира сама ищет по ФИО + контексту; для каждого источника — ник, ключевые метрики (звёзды, карма, репутация, ER).
6. **AI-разведка.** Claude API с web search в режиме **PII-firewall** — в LLM передаются только ФИО, город, должность (без телефона/почты/ИНН). Контракт ответа: цитата + URL + JSON-schema. Находки оформлены как блоки-цитаты с ссылкой на источник.
7. **Алиментные обязательства.** ФССП — реестр должников по алиментам.

### Дизайн-токены
- Цвета источников: `--vf-src-api` (синий), `--vf-src-reg` (фиолетовый), `--vf-src-pub` (оранжевый), `--vf-src-ai` (красный).
- Цвета статусов: `--vf-st-clean` (зелёный), `--vf-st-info` (синий), `--vf-st-warn` (оранжевый), `--vf-st-risk` (красный).
- AI-цитаты — оранжевая полоса слева 3 px, фон `#FFF7F2`, italic.

## 9. Таб «Чат» (`ChatTab`)

Унифицированный чат по всем каналам. Глафира объединяет переписку из TG, hh, WhatsApp, Max, SMS в один поток.

### 9.1 Стрим сообщений (`chat-stream`)
- Day-divider: «08.04.26»
- Строка сообщения:
  - Аватар (для входящих — слева, для исходящих — справа)
  - Над пузырём: имя отправителя + chip канала (точка цвета канала + лейбл)
  - Пузырь сообщения
  - Время справа/слева под пузырём (моно)

Канал отображается у каждого сообщения цветной плашкой (TG бирюзовый, hh красный, WhatsApp зелёный, Max синий, SMS серый, E-mail нейтральный серый `@`).

### 9.2 Compose (`chat-compose`)
- **Селектор канала ответа** (dropdown). Опции в порядке: `Telegram` · `hh.ru` · `Max` · `WhatsApp` · `SMS` · `E-mail`. У каждой опции — цветной круг с короткой меткой (`TG / hh / MX / WA / SMS / @`) и название. По умолчанию — `Telegram`.
- **Textarea** на 2 строки + кнопка отправки (синий paper-plane).
- Подсказка снизу: «Ctrl + Enter — отправить · ответ уйдёт в **<имя канала>**».

Send-кнопка disabled пока пусто.

> E-mail добавлен как последний канал ответа: используется для финальной коммуникации (оффер, подтверждения, формальные документы), когда мессенджеры неуместны.

## 9.7. Таб «Звонки» (`CallsTab`)

Все звонки рекрутёра кандидату, совершённые через телефонию **Манго Телеком** (по номеру кандидата). Визуальная логика — в стиле amoCRM: карточки звонков с плеером, кратким содержанием и AI-разбором.

### 9.7.1 Сводка-строка (сверху)
- `N звонка` · `MM:SS разговора` (суммарное время состоявшихся) · `K недозвон` (красным).
- Справа — плашка телефонии: красная точка + `Манго Телеком · +7 (…)`.

### 9.7.2 Карточка звонка (`call-card`)
Шапка:
- Круглая иконка направления: **исходящий** (синяя, стрелка ↗), **входящий** (зелёная, стрелка ↙), **недозвон** (красная, перечёркнутая).
- Название звонка («Первичный скрининг», «Согласование интервью»…).
- Статус-пилюля: `Исходящий · 4:12` либо красная `Не дозвонился`.
- Справа: имя рекрутёра + дата·время (моно).

### 9.7.3 Аудиоплеер (`call-player`)
Показывается только для состоявшихся звонков (у недозвона записи нет):
- Круглая кнопка play/pause (цвет = направление: синий для исходящих, зелёный для входящих).
- **Волна** из вертикальных баров; пройденная часть заливается цветом. Клик по волне — перемотка (seek).
- Таймер `текущее / общее` (моно).
- Переключатель скорости `1× → 1,5× → 2×`.
- Кнопка скачать запись.

### 9.7.4 Краткое содержание (`call-summary`)
Лейбл «👩🏻 Краткое содержание» + абзац: о чём говорили, к чему пришли. Глафира расшифровывает запись и резюмирует разговор.

### 9.7.5 AI-подсказка (`call-hint`)
Блок-инсайт в стиле amoCRM — что пошло не так / что улучшить:
- **Жёлтый вариант** (`warn`, заголовок «AI-подсказка: что улучшить») — например: рекрутёр говорил 68% времени, не уточнил зарплату/формат, не зафиксировал следующий шаг.
- **Зелёный вариант** (`good`, заголовок «AI-разбор звонка») — удачный звонок + одна рекомендация на улучшение.
- Недозвоны AI-подсказки не имеют (но в кратком содержании отмечается, что Глафира автоматически написала в Telegram).

### 9.7.6 Дизайн-токены
- Точка Манго — фирменный красный `#E2231A` с мягким гало.
- Иконки направления: синий `--ark-blue-*`, зелёный `--ark-green-*`, красный `--ark-red-*`.
- AI-подсказка: жёлтый фон `#FFFBF0` / зелёный `#F2FBF5`, иконка-лампочка.
