# Экран 4. Раздел «Кандидаты» (общая база)

> Открывается при клике на пункт «Кандидаты» (👤) в главном сайдбаре. URL: `/candidates`.
> Это **общая база всех кандидатов компании** — независимо от вакансий. Сюда попадают все, кто когда-либо был добавлен в систему: через отклики, импорт, парсинг расширением, диалог с Анатолием.

## 4A. Список кандидатов

### Шапка раздела (sticky)

- Заголовок «Кандидаты».
- Подзаголовок-счётчик: «1 247 кандидатов в базе» (или «Показано 84 из 1 247» при применённых фильтрах).
- Справа — кнопки: `+ Добавить кандидата` (импорт резюме / ручное создание) · `Импорт из файла`.

### Панель управления (под шапкой, sticky)

**Слева:**
- **Поиск по ФИО** — поле с иконкой-лупой, дебаунс 200мс. Опционально расширенный поиск (по телефону, email, навыкам) — переключатель рядом.

**Справа:**
- **Сортировка** — дропдаун: По дате добавления (новые) · По AI-скорингу · По ФИО А-Я · По дате последней активности.

### Строка фильтров

| Фильтр | Тип | Значения |
|---|---|---|
| **Город** | дропдаун с поиском | Все / конкретный |
| **Опыт** | сегментный контрол | Все · до 1 года · 1–3 · 3–5 · 5+ |
| **AI-скоринг** | сегментный контрол | Все · 🟢 80+ · 🟡 50–79 · 🔴 0–49 |
| **Источник** | дропдаун | Все · hh.ru · Авито · Telegram-Анатолий · Импорт · Ручной ввод |
| **Вакансия** | дропдаун с поиском | Все / конкретная (показываем кандидатов, участвовавших в этой вакансии) |
| **Этап** | дропдаун | Все · Отклик · Отобран · Контакт с рекрутером · Интервью · Контакт с Менеджером · Оффер · Нанят · Отказ · В базе (не привязан) |
| **Теги** | мульти-селект с поиском | Произвольные теги, которые присваивались кандидатам |
| **Дата добавления** | сегментный контрол | Неделя · Месяц · Квартал · Год · Всё время |

Справа в строке фильтров — кнопка **«Сбросить»** (видна, если что-то применено).
Все фильтры синхронизируются с URL (query params) — ссылку можно шарить.

### Тело — сетка карточек

Сетка `auto-fill, минимум ~280px, gap 16px`. На широких экранах 4 в ряд, на средних 2–3, на узких 1.

#### Карточка кандидата

Квадратная (или близкая к квадрату) карточка.

```
┌──────────────────────────────┐
│ [Аватар]   Артём 028-Марков  │
│            🟢 87             │
│                              │
│ 29 лет                       │
│ Аккаунт менеджер B2G         │
│ VK · январь 2026 — апр. 2026│
│                              │
│ ──────────────────────────   │
│ ▣ Региональный менеджер      │
│   🟡 Интервью                │
└──────────────────────────────┘
```

**Содержимое карточки сверху вниз:**

1. **Шапка карточки:**
   - (Опц.) Аватар кандидата слева — маленький круг.
   - **ФИО** справа от аватара — жирным, основной фокус, 1 строка с эллипсисом.
   - **AI-скоринг** в правом верхнем углу — крупный цветной бейдж по градации:
     - 🟢 Зелёный — 80–100
     - 🟡 Жёлтый — 50–79
     - 🔴 Красный — 0–49

2. **Основная инфа** (мелким, плотно):
   - **Возраст:** `29 лет`.
   - **Должность с последнего места работы:** `Аккаунт менеджер B2G`.
   - **Компания · период:** `VK · январь 2026 — апр. 2026`.

   Формат — те же данные, что в столбце «Профиль» табличного списка вакансии: `29 лет / Аккаунт менеджер B2G / VK / январь 2026 — апр. 2026`.

3. **Разделитель** (тонкая линия).

4. **Лейбл вакансии (внизу карточки):**
   - Иконка/маркер + **название вакансии**, на которую кандидат отбирался.
   - **Цветовой индикатор этапа**, на котором кандидат сейчас в этой вакансии. Цвет соответствует этапу из 8-этапной воронки. Может быть цветной пилюлей (chip) или цветной точкой слева от названия этапа.
     - **Нанят** — зелёный
     - **Отказ** — красный
     - Остальные этапы (Отклик / Отобран / Контакт с рекрутером / Интервью / Контакт с Менеджером / Оффер) — нейтральные/брендовые градации.

   **Если кандидат участвовал в нескольких вакансиях** — показываем **последнюю по дате** + бейдж `+N` рядом с лейблом, означающий «ещё в N вакансиях». На hover — список всех вакансий с этапами.

   **Если кандидат ни в одной вакансии** (просто в базе) — лейбл `🔘 В базе · не привязан к вакансии`.

5. **На hover карточки:**
   - Лёгкое поднятие (тень).
   - Появляется иконка `⋯` в углу: *Открыть · Назначить на вакансию · Написать · Объединить дубли · Удалить*.

#### Состояния карточки
- `default` · `hover` (поднятие) · клик → открывает карточку кандидата на весь экран (4B).

### Поведение списка

- Клик по карточке → **4B. Карточка кандидата на весь экран**.
- Клик по AI-скорингу на карточке → открыть карточку с активной вкладкой «Оценка AI».
- Клик по лейблу вакансии внизу карточки → переход на саму вакансию (`/vacancies/:id`).
- Клик по бейджу `+N` (несколько вакансий) → открыть карточку кандидата сразу с раскрытым блоком «История участия».
- Бесконечный скролл с подгрузкой по 24 карточки + скелетон внизу.
- Реалтайм-обновление при добавлении нового кандидата.

### Edge cases списка

- **Нет кандидатов в базе** → empty state: иллюстрация + «В базе пока нет кандидатов» + CTA `+ Добавить вручную` / `Импортировать резюме` / `Подключить Анатолия`.
- **Поиск/фильтры ничего не нашли** → «Никого не найдено по заданным параметрам · Сбросить фильтры».
- **Loading** — 8 карточек-скелетонов.
- **Длинное ФИО / название вакансии** — эллипсис + тултип.
- **Дубли** — если AI определил дубль, на карточке мелкий бейдж `Дубль` в углу. Клик → модалка «Объединить с …».
- **Кандидат без AI-скоринга** — серая плашка `—` вместо цветного бейджа, тултип «Скоринг ещё не посчитан».

## 4B. Карточка кандидата (на весь экран)

> Открывается при клике на карточку кандидата в списке. URL: `/candidates/:candidateId`.
> Главный сайдбар остаётся виден, остальная область — карточка кандидата на всю ширину.

### Лейаут

```
┌────┬─────────────────────────────────────────────────────────┐
│ 🏠 │  ← Назад к кандидатам                                   │
│ 💼 │ ───────────────────────────────────────────────────     │
│ 👤 │                                                         │
│ 📊 │  [Шапка кандидата: ФИО, контакты, ЗП, город, AI-скор]  │
│ 💗 │                                                         │
│ ⚙️ │  [История участия в вакансиях]                          │
│    │                                                         │
│    │  [Вкладки: Резюме · Чат · Документы · Оценка AI]        │
│    │  [Содержимое активной вкладки]                          │
└────┴─────────────────────────────────────────────────────────┘
```

### Шапка кандидата (sticky)

**Кнопка-стрелка `← Назад к кандидатам`** в самом верху — возврат в список с сохранёнными фильтрами и позицией скролла.

**Строка 1 — имя:** `Артём 028-Марков` — крупным, bold.
Справа от имени — действия: `Написать` · `Позвонить` · `Назначить на вакансию` · `⋯` (меню: Скачать резюме, Объединить дубли, Удалить, Добавить тег).

**Строка 2 — основная инфа** (одной строкой через `/`):
`29 лет / Аккаунт менеджер B2G / VK / январь 2026 — апрель 2026`

**Строка 3 — контакты:**
- `+7 (999) 466-20-16` — кликабельный (звонок через IP-телефонию).
- Бейджи мессенджеров рядом — иконки тех мессенджеров, где найден этот номер (Telegram, WhatsApp, Viber). Клик по бейджу → переход на вкладку «Чат» с этим каналом.
- Email (если есть).

**Строка 4 — город:** `Город: Новосибирск (Новосибирская область)`.

**Строка 5 — теги** (мелким): чипы тегов, кнопка `+` для добавления тега.

**AI-скоринг** — крупный цветной бейдж справа от шапки. Клик → переход на вкладку «Оценка AI».

### Блок «История участия в вакансиях»

Под шапкой, перед вкладками. Это **главное отличие карточки в общей базе от карточки внутри вакансии** — здесь нет одной воронки, есть список всех вакансий, в которых участвовал кандидат.

**Заголовок блока:** «История участия в вакансиях (4)».

**Тело блока — список строк, по одной на каждую вакансию:**

```
┌────────────────────────────────────────────────────────────┐
│ ▣ Региональный менеджер по продажам    🟡 Интервью         │
│   Заказчик: Иван Петров · Активна                          │
│   Отбор: 08.04.26 · Скоринг: 🟢 87                         │
│                                                            │
│ ▣ Frontend-разработчик                  🔴 Отказ           │
│   Заказчик: Анна Смирнова · Закрыта                        │
│   Отбор: 12.02.26 → Отказ: 28.02.26                        │
│   Причина отказа: Несоответствие стека                     │
│                                                            │
│ ▣ Менеджер по работе с клиентами        🟢 Нанят           │
│   ...                                                      │
└────────────────────────────────────────────────────────────┘
```

Каждая строка истории содержит:
- **Название вакансии** + статус вакансии (Активна / В архиве).
- **Текущий этап** (или финальный) с цветовой индикацией по 8-этапной воронке.
- **Заказчик** и **ответственный рекрутер** (мелким серым).
- **Дата отбора на вакансию.** Если кандидат на терминальном этапе (Нанят / Отказ) — также дата перехода в этот этап.
- **Скоринг AI на момент отбора** — цветной бейдж.
- **Причина отказа** (если этап = Отказ) — мелким серым.
- На hover строки — иконка-стрелка `→`. Клик → переход в `/vacancies/:id/candidates/:candidateId` (карточка кандидата внутри той вакансии).

**Сортировка истории:** по дате отбора, новые сверху. Активные вакансии можно показать первыми — на усмотрение дизайна.

**Если кандидат ни в одной вакансии** — блок не показывается, вместо него — мелкая плашка `Кандидат пока ни в одной вакансии · Назначить на вакансию`.

### Вкладки (под историей)

| Вкладка | Содержимое |
|---|---|
| **Резюме** | Полный текст резюме кандидата: опыт, образование, навыки, языки. Если резюме — файл (PDF), показываем встроенный просмотрщик + кнопка «Скачать оригинал». Опционально: AI-summary резюме сверху (3–5 строк). |
| **Чат** | Объединённая переписка по всем каналам (Email · Telegram · WhatsApp · Авито Чат). Переключатель канала сверху. Лента сообщений + поле ввода + шаблоны. История диалога Анатолия со значком 🤖. **В общей базе показывается полная история по всем вакансиям**, с разделителями «Контекст: вакансия XYZ». |
| **Документы** | Список файлов кандидата: резюме, фото-док, мед.книжка, согласие на ОПД, сертификаты. Drag-and-drop загрузки. Превью файлов. Каждый файл — название, тип, размер, дата, кто загрузил. |
| **Оценка AI** | Подробная аналитика AI: общий скоринг + breakdown по критериям (опыт, навыки, локация, ЗП, мотивация). Аргументация AI («сильные стороны / риски»). Транскрипт диалога Анатолия. **В общей базе** — отдельные блоки оценки по каждой вакансии, в которой кандидат участвовал. |

Вкладка по умолчанию — **Резюме**.

### Поведение карточки

- Кнопка `← Назад к кандидатам` возвращает в список (4A) с сохранёнными фильтрами и позицией скролла.
- Клик по строке в «Истории участия» → переход в карточку кандидата внутри конкретной вакансии (`/vacancies/:id/candidates/:candidateId`).
- Клик по кнопке `Назначить на вакансию` → модалка с поиском вакансий и выбором этапа.
- Клик по бейджу мессенджера в шапке → открывает вкладку «Чат» с этим каналом.
- Все действия (добавление тега, изменение данных, отправка сообщения) — реалтайм.

### Edge cases карточки

- **Нет резюме** (кандидат пришёл через Анатолия без резюме) → во вкладке «Резюме» показываем структурированные ответы кандидата на скрининге + предложение «Запросить резюме».
- **Кандидат — дубль** → в шапке бейдж `Дубль`, клик → модалка «Объединить с …».
- **Кандидат был удалён или анонимизирован по 152-ФЗ** → карточка в read-only с пометкой и причиной.
- **Loading** — скелетон шапки + истории + вкладок.

## Связь с другими экранами

- Карточка кандидата **внутри вакансии** (`/vacancies/:id/candidates/:candidateId`) и карточка кандидата **в общей базе** (`/candidates/:candidateId`) — **это два разных вида одного и того же кандидата**.
- Внутри вакансии — фокус на одной воронке (Бритрикс24-style), вкладки те же.
- В общей базе — фокус на истории участия по всем вакансиям, вкладки те же, но контент в Чате/Оценке AI агрегирует все вакансии.
- Из любого вида можно перейти в другой (через лейбл вакансии в одну сторону, через кнопку «Открыть в базе» в другую).

## Что осознанно НЕ в этом ТЗ

- Импорт резюме из файла (отдельный flow / модалка).
- Объединение дублей (отдельная модалка).
- Массовые действия (выбор чекбоксами и операции над группой) — добавим позже.
- Расширенный поиск по тексту резюме / навыкам — отдельный режим.
- Управление тегами как сущностью (создание/удаление/переименование) — раздел Настройки.
- Назначение кандидата на вакансию (модалка) — отдельный flow.
