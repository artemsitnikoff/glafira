# Экран 03. Кандидаты на вакансию (воронка)

> Открывается при клике на любую вакансию в подменю «Вакансии». Это **главный рабочий экран рекрутера** — здесь он проводит большую часть времени.

**Файл компонента:** `components/Candidates.jsx` (`CandidatesList`)
**Связанные:** Экран 02 (вакансии), Экран 04 (карточка соискателя)

---

## 1. Назначение

Воронка по конкретной вакансии. На одном экране рекрутер:
- видит всех кандидатов, отобранных Глафирой,
- понимает, на каком этапе кто находится (классическая kanban-воронка, но в виде таблицы со счётчиками),
- быстро находит нужного человека (поиск, сортировки, фильтры),
- открывает карточку любого кандидата прямо здесь же (без перехода на отдельный экран — слайдер вправо),
- массово работает со списком (выбор чекбоксами, перевод/отклонение/сообщение).

## 2. Структура экрана

```
┌──────────────────────────────────────────────────────────────────────┐
│  Шапка вакансии                                                      │
│  • Название · клиент · рекрутер · город · дата создания              │
│  • Действия: Поделиться · Перейти на hh · Редактировать · + Кандидат │
├──────────────────────────────────────────────────────────────────────┤
│  Воронка-чипы (этапы со счётчиками)                                  │
│  Все · Отклик → Отобран → Контакт с рекр. → Интервью → … → Нанят     │
│                                                       │ Отказ        │
├──────────────────────────────────────────────────────────────────────┤
│  Контролы: поиск · (массовые действия) · кнопка «Фильтры»            │
├──────────────────────────────────────────────────────────────────────┤
│  Таблица кандидатов                                                  │
│  ┌──────────────────────────┬─────────────────────────────────────┐  │
│  │ Профиль (закреплён)      │  Остальные колонки (скрол)          │  │
│  │ ☐ Аватар  ФИО  AI-балл   │  Телефон Зарплата Город Дата Этап   │  │
│  └──────────────────────────┴─────────────────────────────────────┘  │
│  При клике на строку → справа выезжает карточка соискателя           │
└──────────────────────────────────────────────────────────────────────┘
```

---

## 3. Шапка вакансии (`vac-header`)

### Левая часть
- **H1**: Название вакансии (например, «Frontend-разработчик (Senior)»).
- **Мета-строка** через разделители «·»: клиент · рекрутер · город · «создана 15 марта 2026».

### Правая часть — действия
1. **Поделиться** (вторичная) — копирует ссылку на воронку для нанимающего менеджера / клиента.
2. **Перейти на hh** (вторичная) — открывает оригинальную вакансию во внешнем источнике.
3. **Редактировать** (вторичная) — модал с настройками вакансии (бриф, режим Глафиры, источники).
4. **+ Добавить кандидата** (primary) — ручное добавление: ФИО + резюме PDF / линк.

---

## 4. Воронка-чипы (`funnel-row`)

Горизонтальная цепочка чипов, по одному на этап. Между чипами — серые стрелки `→`.

### Этапы (в порядке, из `STAGES` в `Candidates.jsx`)
| ID | Лейбл | Цвет точки | Особый |
|---|---|---|---|
| `response` | Отклик | `#5B6573` | — |
| `added` | Добавлен | `#7E5CF0` | system — кандидат добавлен из общей базы; чип в воронке не показывается |
| `selected` | Отобран | `#9AA3AE` | — |
| `recruiter` | Контакт с рекрутером | `#7AB4F5` | — |
| `interview` | Интервью | `#2A8AF0` | — |
| `manager` | Контакт с менеджером | `#5778E8` | — |
| `offer` | Оффер | `#E0A21A` | — |
| `hired` | Нанят | `#16A34A` | terminal — финал, отдельный чип после стрелок |
| `rejected` | Отказ | `#DC4646` | terminal + separated — отделён зазором справа |

Слева отдельный чип **«Все» (N)** — показывает всех, кроме отказов.
После «Нанят» большой пробел `funnel-gap`, потом отдельно красный **«Отказ»**.

### Поведение чипа
- Каждый чип — это фильтр по этапу.
- Активный чип — заливка `--bg-2` + темная рамка + цветная точка.
- Цифра справа — кол-во кандидатов на этом этапе (мелкая, моно).
- Клик закрывает открытую карточку и фильтрует список.

## 5. Контролы (`cand-controls`)

### Поиск
Стандартный input с лупой, ширина ~280 px, ищет по ФИО.

### Bulk-bar (появляется когда выбраны строки)
Слева от кнопки фильтров:
```
[N выбрано]   [Перевести]  [Отклонить ▾]  [Сообщение]
```
Все три кнопки — secondary small. **«Отклонить ▾»** раскрывает popover с двумя группами причин:
- **От кандидата** (`REJECT_REASONS_CAND`): «Не вышел на связь», «Не устроила ЗП», «Принял другой оффер», «Не устроил график», «Слишком далеко от дома».
- **Со стороны компании** (`REJECT_REASONS_CO`): «Несоответствие опыта», «Несоответствие навыков», «Не прошёл интервью», «Не прошёл СБ», «Завышенные ожидания по ЗП».

Причины — пункты-кнопки с цветным буллетом (синий — от кандидата, серый/красный — со стороны компании). Сам справочник причин редактируется в **Настройки → Воронка по умолчанию**.

### Кнопка «Фильтры»
Справа, всегда видна. С badge-числом активных фильтров. Клик открывает **drawer справа** (см. п. 7).

## 6. Таблица кандидатов

### 6.1 Структура колонок
Таблица разделена на 2 части:

**Закреплённая часть «Профиль»** (всегда видна):
- ☐ чекбокс
- Аватар (инициалы цветным кругом)
- **ФИО** + номер кандидата (`#029`) + малая «пипка» цвета этапа
- Возраст / последний опыт (2 строки серым)
- **AI-скоринг** — большая цветная плашка справа

**Скрол-часть «Остальное»** (видна в режиме списка, скрывается когда открыта карточка):
- Телефон + иконки мессенджеров (TG / WA / Max)
- Зарплата (моно)
- Город
- Дата отбора (моно, серым)
- Этап (chip с точкой)

### 6.2 Сортировка колонок
Каждый заголовок-колонка кликабельна. Стрелочки ▲▼ показывают направление. По умолчанию `score desc` (лучшие сверху).

Доступные поля сортировки: ФИО, AI, телефон, зарплата, город, дата, этап, возраст.

### 6.3 AI-скоринг (`ScoreBadge`)
- ≥ 80 — зелёный `score-green`
- 50–79 — жёлтый `score-yellow`
- < 50 — красный `score-red`
- `null` — серый «—»
- Размеры: `sm` / `md` / `lg` / `xl`. В таблице — `lg` (большая плашка справа от ФИО).
- На hover — тултип «Почему такая оценка» (объясняет AI).

### 6.4 Иконки мессенджеров (`MessIconRound`)
Круги фирменных цветов:
- Telegram — `#229ED9` (логотип-самолёт)
- WhatsApp — `#25D366` (логотип-телефон)
- Max — `#0077FF` (буква M)
- Viber — `#7360F2`

Размер в таблице — 18 px, в карточке — 22 px.

### 6.5 Строка кандидата (поведение)
- Клик по строке → открывает карточку справа (Экран 04 как панель).
- Клик по чекбоксу — multi-select без открытия карточки.
- Активная строка (открыта её карточка) — фон `--bg-3` + цветная вертикальная полоска слева цвета этапа (`--stage-color`).
- Hover — фон `--bg-3-hover`.

### 6.6 Пустое состояние
Если на выбранном этапе никого:
> «На этапе «Интервью» пока никого
> Кандидаты появятся, как только Глафира продвинет их по воронке.»

## 7. Drawer фильтров (`FilterDrawer`)

Открывается справа на полную высоту. Затемняющий overlay позади.

### Шапка drawer
- Заголовок «Фильтры»
- Если есть активные — кнопка-кружок «Сбросить» (стрелка-обновление)
- Кнопка X (закрыть)

### Закладка
Кнопка `🔖 Сохранить настроенный фильтр` — disabled, если активных фильтров нет.

### Секции (collapsible)
Все по одинаковому шаблону: заголовок + бэдж количества активных + шеврон.

| Секция | Контрол | По умолчанию |
|---|---|---|
| AI-скоринг | слайдер «от N» (0–100, шаг 5) | 0 |
| Зарплата, тыс ₽ | слайдер «до N» (100–500, шаг 10) | 500 |
| Источник | чипы: HeadHunter / Глафира · Telegram / Авито | — |
| Этап воронки | чипы со всеми этапами и точками | — |
| Город проживания | чипы: Москва / СПб / Новосибирск | — |
| Мессенджер | чипы: Telegram / WhatsApp / Max | — |
| Готовность | чекбокс «Готов к переезду» | off |
| Период отбора | чипы: Сегодня / Неделя / Месяц / Квартал / Всё время | — |
| Повторный отклик | чекбокс | off |

### Подвал drawer
- Слева: `Сбросить всё` (secondary)
- Справа: `Показать N` (primary, число — количество кандидатов после фильтрации)

## 8. Режим открытой карточки (`detailMode`)

Когда пользователь кликнул по строке, справа выезжает **карточка соискателя** (см. Экран 04).

### Что меняется в воронке:
- Колонки «остальное» (телефон/зп/город/дата/этап) **уезжают** — карточка занимает их место.
- Закреплённая часть «Профиль» остаётся видимой слева → можно скролить список и переключаться между кандидатами одним кликом.
- Чипы воронки сверху остаются видимыми. Клик по чипу — закрывает карточку и фильтрует список.
- Активная строка подсвечена (`row.open`).

### Закрытие карточки
- Кнопка X в шапке карточки.
- Клик по чипу воронки.
- Esc.

## 9. Дизайн-токены и стили

- Заголовок вакансии — `--font-display`, 24–28 px, средний вес.
- Мета-строка — `--fg-3`, 13 px.
- Чипы воронки — высота 32 px, `border-radius: 16px`.
- Таблица — высота строки ~64 px (двустрочный профиль).
- Активная строка — `box-shadow: inset 4px 0 0 var(--stage-color)`.
- Цветовая палитра этапов — задана в `STAGES` (см. таблицу п. 4).

---

## 10. Бизнес-логика «что Глафира уже сделала»

В колонке «Дата отбора» — это дата, когда Глафира **сама** добавила кандидата в воронку (а не дата отклика).
Этап «Контакт с рекрутером» = Глафира провела первичный скрининг и передала рекрутеру.
Этап «Контакт с менеджером» = собеседование с нанимающим менеджером назначено.

Глафира может **сама** двигать кандидатов:
- `response → selected` (если AI-скоринг ≥ 80 и нет стоп-факторов)
- `response → rejected` (если стоп-факторы или скоринг < 50)
- `selected → recruiter` (когда инициировала чат и кандидат ответил)
- `recruiter → interview` (когда подтвердила слот в календаре)

Дальше — только рекрутер.
