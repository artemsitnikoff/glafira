# Экран 02. Раздел «Вакансии»

> Открывается при клике на пункт «Вакансии» (📋) в сайдбаре. Главный рабочий раздел рекрутера.

**Файл компонента:** `components/Vacancies.jsx`
**Связанные:** `Sidebar.jsx` (подменю), `Candidates.jsx` (воронка вакансии)

---

## 1. Назначение раздела

Раздел «Вакансии» — это **список всех вакансий в работе** + точка входа в **воронку конкретной вакансии**. Из этого экрана рекрутер:

- видит все свои активные вакансии (в подменю сайдбара),
- открывает воронку по любой из них (один клик → Экран 03),
- создаёт новую вакансию,
- находит архивные/закрытые вакансии,
- настраивает «режимы работы» Глафиры с конкретной вакансией.

---

## 2. Структура раздела

Раздел состоит из **двух колонок**:

```
┌──────────────────────────┬──────────────────────────────────┐
│ Сайдбар                  │  Контент: либо пустой стейт,     │
│  • Главная               │  либо воронка по выбранной       │
│  • Вакансии ▾            │  вакансии (Экран 03),            │
│     ↳ + Новая вакансия   │  либо архив,                     │
│     ↳ поиск              │  либо форма создания/            │
│     ↳ список вакансий    │       редактирования вакансии,   │
│     ↳ Архив              │  либо форма добавления кандидата │
│  • Кандидаты             │                                  │
│  • Аналитика ▾           │                                  │
│  • Пульс-Онбординг       │                                  │
│  • Настройки             │                                  │
└──────────────────────────┴──────────────────────────────────┘
```

Раздел «Вакансии» **не имеет отдельной средней колонки**: список вакансий разворачивается inline внутри основного сайдбара. По клику на вакансию контентная область (правая колонка) подменяется на её воронку (Экран 03).

---

## 3. Раскрытый блок «Вакансии» в сайдбаре

Когда пользователь кликает по пункту «Вакансии» (или по шеврону рядом), под пунктом разворачивается inline-блок (`sub-block`).

### 3.1 Кнопка `+ Новая вакансия` (наверху)
Открывает в контентной области форму создания вакансии (Экран `NewVacancy`). Пункт «Вакансии» в сайдбаре остаётся подсвеченным, конкретная вакансия не выбрана.

### 3.2 Поиск
Компактное поле с иконкой лупы и плейсхолдером «Поиск…». Фильтрует список ниже в реальном времени по подстроке в названии (`name.toLowerCase().includes(query)`).

### 3.3 Список вакансий
Каждый элемент списка:

```
┌─────────────────────────────────────────┐
│ • Frontend-разработчик (Senior)   12 +3 │
└─────────────────────────────────────────┘
```

- Точка слева (`unread-dot`) — индикатор непрочитанных событий по вакансии (если есть).
- Название вакансии.
- Справа — общий счётчик кандидатов (`v.count`) и при наличии — пилюля «новых» откликов (`+N` в брендовом акценте).
- Hover — фон `--bg-3`, выбранная — фон `--bg-active` и левая вертикальная полоска бренд-цветом.
- Если поиск ничего не нашёл — empty-state «Ничего не найдено».

### 3.4 Архив
Под списком — отдельная строка с иконкой коробки и числом архивных вакансий справа. Клик показывает архив в правой колонке (см. п. 4.3).

> Сегментного фильтра статуса (`В работе / Пауза / Архив / Все`) в текущей реализации **нет**: список — это «активные», архив — отдельная строка. Если когда-то понадобятся «На паузе» и «Все», их добавят как сегмент над списком.

---

## 4. Контентная область

В зависимости от того, выбрал ли пользователь вакансию или фильтр:

### 4.1 Без выбранной вакансии (Empty State)
Если раздел «Вакансии» открыт, но конкретная вакансия не выбрана и архив не активен — отображается компонент `EmptyVacancyPane`:
- Иллюстрация-плейсхолдер.
- Заголовок и пояснение, что нужно выбрать вакансию слева или создать новую.

(Тык `defaultSelected` при первом входе в раздел может автоматически выбрать первую вакансию или открыть архив — см. п. 6.)

### 4.2 С выбранной вакансией → Экран 03
Контентная область — воронка кандидатов (компонент `CandidatesList`, см. документ `03 - Кандидаты на вакансию (воронка).md`). Сайдбар слева остаётся видимым.

Внутри воронки можно:
- открыть карточку соискателя (Экран 04) — отдельным компонентом справа от воронки;
- запустить редактирование текущей вакансии — открывается `NewVacancy` в режиме `editMode` с подставленными данными;
- добавить кандидата вручную — открывается `NewCandidate` (см. п. 5).

### 4.3 Архив
Контентная область показывает табличный список закрытых вакансий:

| Колонка | Описание |
|---|---|
| Название | Заголовок вакансии |
| Клиент | Компания-заказчик |
| Рекрутер | Кто вёл |
| Закрыта | Дата закрытия |
| Итог | «Нанят», «Отменена», «Заморожена» |
| Срок | Сколько дней закрывалась (моно) |

## 5. «Режимы работы» Глафиры с вакансией

Это особенность продукта: рекрутер задаёт, **насколько автономно** Глафира ведёт вакансию. Переключатель «режим» лежит в шапке вакансии (на Экране 03), но впервые конфигурируется при создании.

### Три режима

| Режим | Что делает Глафира | Что остаётся человеку |
|---|---|---|
| **Полуавтомат (A)** | Скринит, отклоняет явно нерелевантных, оценивает резюме | Решает по каждому «среднему» кандидату |
| **Автомат (B)** | Скринит, ведёт переписку, назначает интервью | Подтверждает оффер |
| **Под контролем (C)** | Только оценивает резюме | Всё остальное — рекрутер |

Переключатель в шапке: 3 сегмента, текущий — выделен.
По наведению — тултип-объяснение режима.

## 7. Создание новой вакансии (`+ Новая вакансия`)

Клик по кнопке открывает форму **в контентной области** (а не в модалке) — компонент `NewVacancy`. Многошаговая форма с навигацией между разделами:

1. **Описание вакансии** — название, **сортировка** (число, по умолчанию 500 — определяет позицию вакансии в списке слева сайдбара; меньше число = выше), город, дедлайн, число позиций, отдел, тип занятости, конфиденциальность, вилка `от/до`, тело описания.
2. **Воронка** — выбор шаблона воронки (По умолчанию / Массовый подбор / Техническая с тестовым / Продажи) и редактирование её этапов (старт · промежуточные · финальные).
3. **Команда** — мульти-выбор сотрудников (`team: ['u1', 'u3', ...]`), которые ведут вакансию. Первый добавленный — ответственный.
4. **Автоматизация** — режимы Глафиры на конкретных этапах: автоперевод (`autoMove` + порог), автоквалификация диапазоном этапов (`autoQA` от и до), автоотказ (`autoReject`).

> **Поле «Сортировка»** — отдельная строка под названием вакансии, узкий числовой инпут (моно, шаг 10, дефолт `500`). Помогает рекрутёру вручную упорядочить вакансии в сайдбаре по приоритету. С подписью «(порядок в списке)» под лейблом.

Та же форма используется в режиме **редактирования** (`editMode={true}`, `initial={…}`) при клике «Редактировать вакансию» из шапки воронки. После сохранения — возврат в воронку.

## 8. Добавление кандидата вручную

Из воронки (Экран 03) можно нажать «+ Кандидат» — открывается компонент `NewCandidate` в контентной области. Форма привязана к текущей вакансии (`vacancyId`). После создания — возврат в воронку.

### Поля формы

| Поле | Тип | Обязательное |
|---|---|---|
| Добавить в вакансию | dropdown | да (предзаполнено `vacancyId`) |
| **Источник** | dropdown | **да** — отдельным полем под выбором вакансии, с подсказкой «· откуда узнали о кандидате». Опции: `HeadHunter` · `Avito Работа` · `SuperJob` · `Telegram-канал` · `Реферальная программа` · `Прямой контакт` · `Кадровое агентство` · `Другое` |
| Фамилия / Имя | text | да |
| Отчество | text | нет |
| Телефон | masked input | нет (но один из контактов нужен) |
| E-mail | email | нет |
| Пол / ДР / Город | segmented + date + text | нет |
| Ожидаемая ЗП + валюта | number + dropdown (`руб.` / `$` / `€`) | нет |
| Тип добавления | dropdown (`Ручное` / `Из резюме` / `Из общей базы` / `По ссылке HH`) | да (дефолт `manual`) |
| Социальные сети | dropdown типа + url | нет |
| Комментарий | textarea | нет |
| Резюме (drop-zone) / Excel-импорт | файл | нет — но если приложить, Глафира распарсит и подтянет недостающие поля |

Кнопка `+ Добавить кандидата` активна, только если заполнены **Фамилия + Имя + Источник**.

## 9. Интеракции

| Действие | Результат |
|---|---|
| Клик по вакансии в раскрытом блоке сайдбара | Открывается воронка (Экран 03) |
| Клик `+ Новая вакансия` | Открывается форма создания в контентной области |
| Поиск в блоке | Фильтрует список в реальном времени |
| Клик «Архив» в блоке | Архив в контентной области |
| Клик «Редактировать» в шапке воронки | `NewVacancy` в режиме редактирования |
| Клик `+ Кандидат` в воронке | `NewCandidate` |

## 10. Дизайн-детали

- Раскрытый блок `sub-block` живёт **внутри** сайдбара (никакой отдельной колонки), фон тот же `--bg-2`.
- Активная вакансия — фон `--bg-active` + левая вертикальная полоска бренд-цветом.
- Счётчики (`sub-count`) — серый моно `--font-mono`, ~12 px.
- Пилюля «новых» (`sub-new`) — компактная плашка бренд-акцентом (`+N`).
- Кнопка `+ Новая вакансия` (`sub-add`) — широкая текстовая кнопка с иконкой плюса наверху блока.

## 11. Связи с другими экранами

- **Сайдбар (Экран 10)** — содержит раскрывающийся блок «Вакансии».
- **Экран 03 (Воронка)** — открывается при клике на любую вакансию.
- **Экран 04 (Карточка соискателя)** — открывается из воронки.
- **Экран 06 (Аналитика)** — там есть пресеты по конкретным вакансиям.
